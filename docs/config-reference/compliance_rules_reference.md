## 1. sensitive_words — 敏感词列表

### 概述

敏感词检测是合规检查的第一道防线。M-12 合规性自检模块会扫描文案中是否包含列表中的任何敏感词，包含即标记为不合规。

### 字段说明

| 属性 | 说明 |
|------|------|
| **类型** | string[] |
| **必填** | 是 |
| **用途** | 广告法违禁词、极限用语检测 |
| **检查方式** | 精确匹配 + 子串匹配 |

### 当前敏感词详解

#### `"第一"`

| 属性 | 说明 |
|------|------|
| **规则分类** | 广告法 — 绝对化用语 |
| **触发条件** | 文案中包含"第一" |
| **法律依据** | 《广告法》第九条第三项：禁止使用"最高级""最佳"等绝对化用语 |
| **合规等级** | CMP-002（广告法合规） |
| **例外情况** | 有权威数据来源可证明时可豁免 |
| **修改建议** | 改为"领先""前列""位居前茅"等相对表述 |

#### `"最好"`

## 2. competitor_names — 竞品名称检测

### 字段说明

| 属性 | 说明 |
|------|------|
| **类型** | string[] |
| **必填** | 否 |
| **默认值** | `[]`（空列表） |
| **用途** | 检测文案中是否提及竞品名称，避免无意中为竞品引流 |
| **检查方式** | 精确匹配 |
| **触发动作** | 标记为警告，不阻塞发布，但需人工确认 |

### 配置示例

```yaml
competitor_names:
  - "竞品A"
  - "竞品B"
  - "某品牌官方名称"
```

### 规则说明

- 此列表为空时，不执行竞品检测
- 检测到竞品名称时，M-12 会生成 `CMP-005` 检查项，`passed=false` 但不会阻塞整体通过
- 需要在 M-13 审阅环节由人工确认是否为有意提及

## 3. platform_rules — 平台特定规则

### 3.1 douyin（抖音）

#### `platform_rules.douyin.max_video_duration_sec`

| 属性 | 说明 |
|------|------|
| **类型** | integer |
| **必填** | 是 |
| **默认值** | `900` |
| **说明** | 抖音允许的最大视频时长（秒），即 15 分钟 |
| **检查方式** | 视频文件实际时长 vs 此值 |
| **超限处理** | 标记为不合规，建议剪辑缩短 |

#### `platform_rules.douyin.recommended_ratio`

| 属性 | 说明 |
|------|------|
| **类型** | string |
| **必填** | 是 |
| **默认值** | `"9:16"` |
| **说明** | 抖音推荐的视频/封面宽高比。9:16 为竖屏全屏 |
| **检查方式** | 实际分辨率计算宽高比 vs 推荐值 |
| **偏差处理** | 偏差 > 5% 时标记为警告，不阻塞发布 |

### 3.2 xiaohongshu（小红书）

#### `platform_rules.xiaohongshu.max_images`

| 属性 | 说明 |
|------|------|
| **类型** | integer |
| **必填** | 是 |
| **默认值** | `18` |
| **说明** | 小红书单篇笔记最多可上传的图片数量 |
| **检查方式** | 实际图片数量 vs 此值 |
| **超限处理** | 标记为不合规，建议删减图片 |

#### `platform_rules.xiaohongshu.recommended_ratio`

| 属性 | 说明 |
|------|------|
| **类型** | string |
| **必填** | 是 |
| **默认值** | `"3:4"` |
| **说明** | 小红书推荐的封面图片宽高比 |
| **检查方式** | 实际分辨率计算宽高比 vs 推荐值 |

### 3.3 bilibili（B站）

#### `platform_rules.bilibili.max_video_duration_sec`

| 属性 | 说明 |
|------|------|
| **类型** | integer |
| **必填** | 是 |
| **默认值** | `3600` |
| **说明** | B站允许的最大视频时长（秒），即 60 分钟（普通用户） |
| **注意** | 大会员或认证用户可能有更长的时长限制 |

#### `platform_rules.bilibili.recommended_ratio`

| 属性 | 说明 |
|------|------|
| **类型** | string |
| **必填** | 是 |
| **默认值** | `"16:9"` |
| **说明** | B站推荐的视频宽高比。16:9 为横屏标准 |
| **检查方式** | 实际分辨率计算宽高比 vs 推荐值 |

### 3.4 twitter（X / Twitter）

#### `platform_rules.twitter.max_text_length`

| 属性 | 说明 |
|------|------|
| **类型** | integer |
| **必填** | 是 |
| **默认值** | `280` |
| **说明** | X（Twitter）单条推文最大字符数 |
| **注意** | X Premium 用户可发最长 25000 字符 |
| **检查方式** | 文案字符数 vs 此值 |
| **超限处理** | 标记为不合规，建议缩短或拆分为线程 |

#### `platform_rules.twitter.recommended_ratio`

| 属性 | 说明 |
|------|------|
| **类型** | string |
| **必填** | 是 |
| **默认值** | `"1:1"` |
| **说明** | X（Twitter）推荐的图片/视频宽高比。1:1 为正方形 |
| **注意** | 也支持 16:9 和 9:16，但 1:1 在时间线中展示效果最佳 |

## 4. 合规检查规则 ID 映射

M-12 合规性自检模块使用以下规则 ID 与配置对应：

| 规则 ID | 规则名称 | 对应配置 | 检查内容 |
|---------|---------|---------|---------|
| CMP-001 | 敏感词检测 | `sensitive_words` | 扫描文案中的敏感词 |
| CMP-002 | 广告法合规 | `sensitive_words` | 广告法极限用语检测 |
| CMP-003 | 版权素材检查 | Material.copyright_status | 引用素材版权状态 |
| CMP-004 | 平台规则检查 | `platform_rules` | 平台特定格式/时长限制 |
| CMP-005 | 竞品名称检测 | `competitor_names` | 文案中是否提及竞品 |

## 5. 规则扩展指南

### 添加新的敏感词

```yaml
sensitive_words:
  # 已有规则...
  - "新敏感词1"
  - "新敏感词2"
```

添加后无需重启，M-12 下次执行时自动生效。

### 添加新的竞品名称

```yaml
competitor_names:
  - "竞品品牌A"
  - "竞品品牌B"
```

### 调整平台规则

```yaml
platform_rules:
  douyin:
    max_video_duration_sec: 1800  # 调整到30分钟
    recommended_ratio: "9:16"
```

## 6. 合规检查流程

```
内容定稿 (M-11)
      │
      ▼
合规性自检 (M-12)
      │
      ├── CMP-001: 敏感词扫描 ──────► 命中 → overall_passed = false
      │                                     → 返回修改建议
      │
      ├── CMP-002: 广告法合规 ──────► 命中 → overall_passed = false
      │                                     → 返回修改建议
      │
      ├── CMP-003: 版权检查 ────────► 异常 → overall_passed = false
      │
      ├── CMP-004: 平台规则 ────────► 不合规 → overall_passed = false
      │
      └── CMP-005: 竞品检测 ────────► 命中 → 警告（不阻塞）
                                          → 人工确认
      │
      ▼
ComplianceResult → 返回 M-11
  ├── overall_passed = true  → 进入审阅流程 (M-13)
  └── overall_passed = false → 返回修改 (M-11)
```
