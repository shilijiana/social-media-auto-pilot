# M-14: 版本变更管理

## 1. 模块概述

| 属性 | 值 |
|------|-----|
| 模块编号 | M-14 |
| 模块名称 | 版本变更管理 |
| 所属类别 | 审阅管理 |
| 优先级 | P1（高优先级） |
| 核心职责 | 为每条内容构建版本树（草稿→V1→V2→...→终稿），在每次修改时自动记录变更详情（修改位置、修改前后内容、变更原因），在提交审阅时生成"版本变更说明"，支持版本回滚操作 |
| 目标用户 | 内容创作者、审阅者、Hermes Agent |

## 2. 输入输出定义

### 2.1 输入

| 输入项 | 来源模块 | 数据格式 | 触发条件 |
|--------|----------|----------|----------|
| 内容修改操作 | M-11（内容编辑） | `ContentEditEvent` | 用户/Hermes 对内容执行编辑操作 |
| 审阅退回请求 | M-13（内容审阅） | `ReviewRejectEvent` | 审阅者退回内容并附修改意见 |
| 版本回滚指令 | 前端/Hermes | `RollbackCommand` | 用户或 Agent 发起回滚请求 |

### 2.2 输出

| 输出项 | 目标 | 数据格式 | 说明 |
|--------|------|----------|------|
| 版本记录 | `content_versions` 表 | `VersionRecord` | 持久化的版本快照 |
| 变更日志文档 | `change_logs` 表 / Markdown 文档 | `ChangeLog` | 结构化的变更记录 |
| 版本变更说明 | M-13（审阅参考） | `VersionChangeSummary` | 审阅提交时自动生成 |
| 回滚结果 | 前端通知 | `RollbackResult` | 回滚操作的状态与详情 |

## 4. 核心业务逻辑

### 4.1 版本创建流程

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  内容变更事件  │───▶│  计算差异     │───▶│  创建版本节点  │───▶│  记录变更详情  │
│  (M-11/M-13) │    │  (diff)      │    │  (version)   │    │  (changelog) │
└──────────────┘    └──────────────┘    └──────────────┘    └──────┬───────┘
                                                                    │
                                                                    ▼
                                                             ┌──────────────┐
                                                             │  生成变更摘要  │
                                                             │  (summary)   │
                                                             └──────────────┘
```

### 4.2 差异计算算法

- **文本差异**: 使用 Myers diff 算法，精确到字符级
- **媒体差异**: 基于 media_id 集合进行集合差运算
- **标签差异**: 基于 hashtag 数组进行集合差运算
- **元数据差异**: 对 metadata JSON 对象进行深度比较

### 4.3 版本树结构

```
                草稿 (V1)
                  │
            ┌─────┼─────┐
            ▼           ▼
           V2          V2' (回滚后分支)
            │
       ┌────┼────┐
       ▼         ▼
      V3       V3' (并行编辑分支)
       │
       ▼
     终稿 (Final)
```

### 4.4 版本回滚逻辑

1. 验证目标版本存在且状态为 `active`
2. 将当前内容替换为目标版本的 `content_snapshot`
3. 创建新版本节点（不覆盖历史），标注 `change_source: "rollback"`
4. 生成回滚变更日志，说明从哪个版本回滚到哪个版本
5. 返回 `RollbackResult` 包含操作状态和新版本号

### 4.5 审阅提交时的变更说明生成

1. 获取上次审阅版本与当前版本的差异
2. 聚合所有 `ChangeLog` 记录
3. 按变更类型分类汇总
4. 生成 Markdown 格式的 `VersionChangeSummary`
5. 作为审阅提交的附件传递给 M-13

## 5. 接口定义

### 5.1 输入接口

| 接口名称 | 调用方 | 描述 |

## 6. 数据库表设计

### 6.1 content_versions 表

```sql
CREATE TABLE content_versions (
    version_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    content_id UUID NOT NULL REFERENCES contents(content_id) ON DELETE CASCADE,
    version_number INTEGER NOT NULL,
    version_label VARCHAR(20) NOT NULL DEFAULT 'draft',
    parent_version_id UUID REFERENCES content_versions(version_id),
    content_snapshot JSONB NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    created_by VARCHAR(100) NOT NULL,
    change_summary TEXT,
    status VARCHAR(20) NOT NULL DEFAULT 'active',
    UNIQUE(content_id, version_number)
);

CREATE INDEX idx_content_versions_content_id ON content_versions(content_id);
CREATE INDEX idx_content_versions_parent ON content_versions(parent_version_id);
```

### 6.2 change_logs 表

```sql
CREATE TABLE change_logs (
    change_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    version_id UUID NOT NULL REFERENCES content_versions(version_id) ON DELETE CASCADE,
    content_id UUID NOT NULL REFERENCES contents(content_id) ON DELETE CASCADE,
    change_type VARCHAR(30) NOT NULL,
    change_location JSONB NOT NULL,
    before_value TEXT,
    after_value TEXT,
    change_reason TEXT,
    change_source VARCHAR(30) NOT NULL,
    review_comment_id UUID,
    changed_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    changed_by VARCHAR(100) NOT NULL
);

CREATE INDEX idx_change_logs_version ON change_logs(version_id);
CREATE INDEX idx_change_logs_content ON change_logs(content_id);
```

### 6.3 version_change_summaries 表

```sql
CREATE TABLE version_change_summaries (
    summary_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    content_id UUID NOT NULL REFERENCES contents(content_id) ON DELETE CASCADE,
    from_version INTEGER NOT NULL,
    to_version INTEGER NOT NULL,
    changes JSONB NOT NULL,
    total_changes INTEGER NOT NULL,
    generated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    generated_by VARCHAR(100) NOT NULL
);

CREATE INDEX idx_summaries_content ON version_change_summaries(content_id);
```

## 7. 异常处理

| 异常场景 | 处理策略 |
|----------|----------|
| 版本快照过大（>10MB） | 仅存储 diff 增量，完整快照存储至对象存储 |
| 并发版本创建冲突 | 乐观锁：基于 version_number + content_id 唯一约束，冲突时重试 |
| 回滚目标版本不存在 | 返回错误码 `VERSION_NOT_FOUND`，提示用户选择有效版本 |
| 差异计算超时 | 设定 30 秒超时，超时后降级为全量快照存储 |
| 变更日志写入失败 | 重试 3 次，仍失败则记录到错误队列，不阻塞版本创建 |

## 8. Hermes Agent 执行指令

```yaml
hermes_agent:
  module: M-14
  triggers:
    - event: content.edit.committed
      source: M-11
      action: create_version_from_edit
    - event: review.rejected
      source: M-13
      action: create_version_from_review_feedback
    - command: rollback_version
      source: [user, hermes]
      action: execute_rollback
    - command: generate_change_summary
      source: M-13
      action: generate_summary_for_review

  workflows:
    create_version_from_edit:
      steps:
        - step: 获取当前最新版本快照
          function: get_latest_version(content_id)
          on_error: abort
        - step: 计算差异
          function: compute_diff(old_snapshot, new_content)
          timeout: 30s
          on_error: fallback_to_full_snapshot
        - step: 创建版本节点
          function: insert_version(content_id, version_number + 1, new_snapshot, parent_version_id)
          on_error: retry(3)
        - step: 记录变更日志
          function: insert_change_logs(version_id, diff_results, change_source, changed_by)
          on_error: retry(3) >> error_queue
        - step: 更新版本变更摘要缓存
          function: update_summary_cache(content_id)
        - step: 通知订阅者
          function: notify("version.created", { content_id, version_id, version_number })

    create_version_from_review_feedback:
      steps:
        - step: 关联审阅意见
          function: link_review_comments(content_id, review_id)
        - step: 创建版本（同上流程）
          function: create_version_from_edit(content_id, edit_event, change_source="review_feedback")
        - step: 在变更日志中标注 review_comment_id
          function: tag_changes_with_review(changes, review_id)

    execute_rollback:
      preconditions:
        - condition: 目标版本存在
          check: version_exists(content_id, target_version_id)
          fail_message: "目标版本不存在"
        - condition: 目标版本状态为 active
          check: version_status(target_version_id) == "active"
          fail_message: "目标版本已被归档或已回滚"
      steps:
        - step: 获取目标版本快照
          function: get_version_snapshot(target_version_id)
        - step: 将目标快照写入当前内容
          function: overwrite_content(content_id, target_snapshot)
        - step: 创建新版本节点（标注回滚来源）
          function: insert_version(content_id, next_version, target_snapshot, parent=current_version, label="rollback")
        - step: 记录回滚变更日志
          function: insert_change_logs(version_id, [{type:"rollback", from:current_version, to:target_version}], "rollback", operator)
        - step: 返回回滚结果
          function: respond({ status: "success", new_version: next_version, rolled_from: current_version, rolled_to: target_version })

    generate_summary_for_review:
      steps:
        - step: 确定版本范围
          function: get_version_range(content_id)
        - step: 聚合变更日志
          function: aggregate_changes(content_id, from_version, to_version)
        - step: 按类别分组
          function: group_changes_by_type(changes)
        - step: 生成 Markdown 文档
          function: format_change_summary_markdown(grouped_changes, from_version, to_version)
          template: |
            # 版本变更说明
            ## 基本信息
            - 内容ID: {content_id}
            - 版本范围: V{from_version} → V{to_version}
            - 变更总数: {total_changes}

            ## 变更详情
            ### 文本修改 ({text_changes_count})
            {text_changes_list}

            ### 媒体变更 ({media_changes_count})
            {media_changes_list}

            ### 标签变更 ({hashtag_changes_count})
            {hashtag_changes_list}

            ### 元数据变更 ({metadata_changes_count})
            {metadata_changes_list}
        - step: 持久化变更说明
          function: save_change_summary(content_id, from_version, to_version, summary_markdown)
        - step: 传递给 M-13
          function: send_to_review(summary_id, content_id)
```

## 9. 非功能性需求

| 类别 | 要求 |
|------|------|
| 性能 | 差异计算 < 5 秒（10KB 文本）；版本创建 < 2 秒 |
| 可靠性 | 版本数据持久化成功率 > 99.9% |
| 可扩展性 | 支持单条内容最多 1000 个版本 |
| 安全 | 版本回滚需权限校验（仅内容所有者和管理员） |
| 审计 | 所有版本操作完整记录，不可篡改 |
