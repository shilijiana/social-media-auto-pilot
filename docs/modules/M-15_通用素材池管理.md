# M-15: 通用素材池管理

## 1. 模块概述

| 属性 | 值 |
|------|-----|
| 模块编号 | M-15 |
| 模块名称 | 通用素材池管理 |
| 所属类别 | 审阅管理 |
| 优先级 | P2（中优先级） |
| 核心职责 | 维护独立的通用素材池（片头动画、Logo、库存BGM等），标记"可复用"属性使其不被按主题重复采集，支持人工增删改，系统自动推荐高频使用素材进入通用池 |
| 目标用户 | 内容创作者、素材管理员、Hermes Agent |

## 2. 输入输出定义

### 2.1 输入

| 输入项 | 来源模块 | 数据格式 | 触发条件 |
|--------|----------|----------|----------|
| 素材使用记录 | M-06（素材管理） | `MaterialUsageRecord` | 每次素材被使用时触发 |
| 人工添加素材 | 前端管理界面 | `AddMaterialCommand` | 管理员手动添加通用素材 |
| 人工删除素材 | 前端管理界面 | `RemoveMaterialCommand` | 管理员手动移除通用素材 |
| 人工修改素材 | 前端管理界面 | `UpdateMaterialCommand` | 管理员修改素材属性 |
| 推荐审核决策 | 前端管理界面 | `RecommendationDecision` | 管理员审核系统推荐 |

### 2.2 输出

| 输出项 | 目标 | 数据格式 | 说明 |
|--------|------|----------|------|
| 通用素材记录 | `common_materials` 表 | `CommonMaterial` | 持久化的通用素材数据 |
| 推荐候选列表 | 前端通知 | `RecommendationList` | 推荐进入通用池的素材列表 |
| 素材优先级标记 | M-07（素材调度） | `PriorityFlag` | 标记素材为通用池来源，M-07 优先使用 |

## 4. 核心业务逻辑

### 4.1 通用素材生命周期

```
┌──────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  素材创建  │───▶│  普通素材池    │───▶│  使用频率追踪  │───▶│  推荐候选评估  │
│  (M-06)   │    │  (M-06)      │    │  (M-15)      │    │  (M-15)      │
└──────────┘    └──────────────┘    └──────────────┘    └──────┬───────┘
                                                               │
                                          ┌────────────────────┘
                                          ▼
                                   ┌──────────────┐    ┌──────────────┐
                                   │  人工审核决策  │───▶│  通用素材池    │
                                   │  (管理员)     │    │  (M-15)      │
                                   └──────────────┘    └──────┬───────┘
                                                               │
                                                               ▼
                                                        ┌──────────────┐
                                                        │  M-07 优先调度 │
                                                        │  (素材调度)   │
                                                        └──────────────┘
```

### 4.2 推荐算法

系统按以下公式计算推荐评分：

```
Score = w1 × FrequencyScore + w2 × CrossTopicScore + w3 × RecencyScore + w4 × QualityScore

默认权重: w1=0.35, w2=0.30, w3=0.20, w4=0.15
```

- **使用频率得分 (FrequencyScore)**: `min(total_uses / threshold, 1.0) × 100`，阈值默认 20
- **跨主题得分 (CrossTopicScore)**: `min(unique_topics / 3, 1.0) × 100`，至少 3 个主题
- **近期活跃度 (RecencyScore)**: 30 天内使用次数 / 10 × 100，上限 100
- **质量得分 (QualityScore)**: 基于素材元数据（分辨率、时长、格式等）评估

推荐阈值: Score ≥ 60 自动进入推荐候选列表。

### 4.3 人工管理操作

| 操作 | 前置条件 | 执行逻辑 |
|------|----------|----------|
| 添加素材 | 无 | 创建 `CommonMaterial` 记录，`source=manual` |
| 修改素材 | 素材存在且状态非 `archived` | 更新 `CommonMaterial` 属性，记录操作日志 |
| 删除素材 | 素材存在 | 标记 `status=archived`（软删除），检查是否有内容引用 |
| 审批推荐 | 推荐候选状态为 `pending` | 更新 `status=approved`，创建 `CommonMaterial` 记录 |

### 4.4 M-07 集成逻辑

当 M-07（素材调度）请求素材时：
1. M-07 发送素材需求（类型、标签、时长等）
2. M-15 返回通用池中匹配的素材（优先返回）
3. 若通用池无匹配，M-07 回退到普通素材池（M-06）
4. 返回结果中标注 `source: "common_pool"` 或 `source: "regular_pool"`

## 6. 数据库表设计

### 6.1 common_materials 表

```sql
CREATE TABLE common_materials (
    material_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(255) NOT NULL,
    material_type VARCHAR(50) NOT NULL,
    description TEXT,
    storage_url TEXT NOT NULL,
    thumbnail_url TEXT,
    file_size BIGINT,
    duration_seconds DOUBLE PRECISION,
    format VARCHAR(20),
    tags TEXT[] DEFAULT '{}',
    usage_count INTEGER DEFAULT 0,
    last_used_at TIMESTAMPTZ,
    added_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    added_by VARCHAR(100) NOT NULL,
    source VARCHAR(30) NOT NULL DEFAULT 'manual',
    status VARCHAR(30) NOT NULL DEFAULT 'active',
    custom_metadata JSONB DEFAULT '{}'
);

CREATE INDEX idx_cm_type ON common_materials(material_type);
CREATE INDEX idx_cm_status ON common_materials(status);
CREATE INDEX idx_cm_tags ON common_materials USING GIN(tags);
CREATE INDEX idx_cm_usage ON common_materials(usage_count DESC);
```

### 6.2 material_usage_records 表

```sql
CREATE TABLE material_usage_records (
    record_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    material_id UUID NOT NULL,
    material_name VARCHAR(255) NOT NULL,
    material_type VARCHAR(50) NOT NULL,
    content_id UUID NOT NULL,
    content_topic VARCHAR(100),
    used_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    used_by VARCHAR(100) NOT NULL
);

CREATE INDEX idx_mur_material ON material_usage_records(material_id);
CREATE INDEX idx_mur_used_at ON material_usage_records(used_at);
CREATE INDEX idx_mur_content ON material_usage_records(content_id);
```

### 6.3 recommendation_candidates 表

```sql
CREATE TABLE recommendation_candidates (
    candidate_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    material_id UUID NOT NULL,
    material_name VARCHAR(255) NOT NULL,
    material_type VARCHAR(50) NOT NULL,
    recommendation_score DECIMAL(5,2) NOT NULL,
    score_breakdown JSONB NOT NULL,
    usage_stats JSONB NOT NULL,
    recommended_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    status VARCHAR(20) NOT NULL DEFAULT 'pending',
    reviewed_by VARCHAR(100),
    reviewed_at TIMESTAMPTZ,
    review_comment TEXT
);

CREATE INDEX idx_rc_status ON recommendation_candidates(status);
CREATE INDEX idx_rc_score ON recommendation_candidates(recommendation_score DESC);
```

## 7. 异常处理

| 异常场景 | 处理策略 |
|----------|----------|
| 素材文件不存在（URL 失效） | 标记 `status=archived`，通知管理员 |
| 重复添加相同素材 | 基于 storage_url 做去重检测，提示"素材已存在" |
| 删除被引用的素材 | 软删除（archived），M-07 调度时过滤已归档素材 |
| 推荐评分计算超时 | 使用缓存的计算结果，异步更新评分 |
| 素材池超过容量上限（默认 10000） | 触发告警，提示清理低频素材 |

## 8. Hermes Agent 执行指令

```yaml
hermes_agent:
  module: M-15
  triggers:
    - event: material.used
      source: M-06
      action: record_usage_and_evaluate
    - command: add_common_material
      source: [user, admin]
      action: execute_add
    - command: update_common_material
      source: [user, admin]
      action: execute_update
    - command: remove_common_material
      source: [user, admin]
      action: execute_remove
    - command: approve_recommendation
      source: [user, admin]
      action: execute_approval
    - schedule: "0 2 * * *"
      action: run_recommendation_engine

  workflows:
    record_usage_and_evaluate:
      steps:
        - step: 记录使用日志
          function: insert_usage_record(usageRecord)
          on_error: retry(3) >> error_queue
        - step: 更新素材使用计数
          function: update_usage_count(material_id, increment=1)
        - step: 检查推荐条件
          function: check_recommendation_eligibility(material_id)
          conditions:
            - if: total_uses >= 10 AND unique_topics >= 2
              then: queue_for_recommendation_evaluation(material_id)

    run_recommendation_engine:
      schedule: daily at 02:00
      steps:
        - step: 查询素材使用统计
          function: get_material_usage_stats(days=90)
        - step: 过滤已在通用池的素材
          function: exclude_common_pool_materials(stats)
        - step: 计算推荐评分
          function: calculate_recommendation_scores(filtered_stats)
          weights: { frequency: 0.35, cross_topic: 0.30, recency: 0.20, quality: 0.15 }
        - step: 筛选达标候选
          function: filter_candidates(scored_list, threshold=60)
        - step: 写入推荐候选表
          function: upsert_recommendation_candidates(candidates)
        - step: 通知管理员
          function: notify_admin("new_recommendations", { count: len(candidates) })

    execute_add:
      preconditions:
        - condition: 素材文件可访问
          check: verify_storage_url(materialData.storage_url)
          fail_message: "素材文件不可访问"
        - condition: 素材不重复
          check: not exists_by_url(materialData.storage_url)
          fail_message: "该素材已存在于通用池中"
      steps:
        - step: 上传缩略图（如有）
          function: generate_thumbnail(materialData.storage_url)
          optional: true
        - step: 创建通用素材记录
          function: insert_common_material(materialData, source="manual")
        - step: 记录操作日志
          function: log_operation("add", material_id, operator)
        - step: 返回结果
          function: respond({ status: "success", material_id })

    execute_update:
      preconditions:
        - condition: 素材存在且未归档
          check: material.status != "archived"
          fail_message: "素材不存在或已归档"
      steps:
        - step: 更新素材属性
          function: update_common_material(material_id, updates)
        - step: 记录操作日志
          function: log_operation("update", material_id, operator, updates)

    execute_remove:
      steps:
        - step: 检查引用关系
          function: check_material_references(material_id)
        - step: 软删除素材
          function: archive_material(material_id)
        - step: 若存在引用，通知受影响内容
          function: notify_referenced_contents(material_id, references)
        - step: 记录操作日志
          function: log_operation("remove", material_id, operator)

    execute_approval:
      steps:
        - step: 更新推荐候选状态
          function: update_candidate_status(candidate_id, decision, reviewer)
        - step: 若批准，创建通用素材记录
          if: decision == "approved"
          function: insert_common_material_from_candidate(candidate_id)
        - step: 若拒绝，记录拒绝原因
          if: decision == "rejected"
          function: record_rejection_reason(candidate_id, reason)
        - step: 通知审核结果
          function: notify_approval_result(candidate_id, decision)

    query_for_m07:
      description: 为 M-07 提供通用池素材查询
      steps:
        - step: 解析查询条件
          function: parse_material_query(filters)
        - step: 查询通用池匹配素材
          function: search_common_materials(type, tags, limit)
          priority: 通用池优先
        - step: 若通用池结果不足，查询普通池
          function: search_regular_materials(type, tags, remaining_limit)
          condition: len(common_results) < limit
        - step: 合并结果并标注来源
          function: merge_and_tag_results(common_results, regular_results)
        - step: 返回素材列表
          function: respond({ materials: merged_results })
```

## 9. 非功能性需求

| 类别 | 要求 |
|------|------|
| 性能 | 素材查询响应 < 200ms；推荐引擎每日批处理 < 5 分钟 |
| 可靠性 | 使用记录写入成功率 > 99.9% |
| 可扩展性 | 通用池容量上限 10000 个素材；支持按类型水平分表 |
| 安全 | 管理操作需管理员权限；素材文件访问需签名 URL |
| 审计 | 所有管理操作记录完整操作日志 |
