```
M-03 位置说明:
  上游: M-01（关键词智能扩展） → 传入扩展关键词用于搜索视频
  下游: M-05（素材去重与质量控制）、M-06（素材入库与版权追踪）、M-07（素材匹配与调用）、M-10（视频初剪）
  自学习: M-25 收集 BGM 风格偏好和剪辑节奏偏好数据
```

### 1.2 核心职责

使用扩展关键词从免费视频素材库（Pexels Video、Pixabay Video）搜索并下载视频片段（b-roll、风景、场景素材）和背景音乐/音效。对视频进行时长过滤、分辨率过滤、方向筛选，对音频进行情绪/节奏标注。输出素材记录写入 `materials` 表，文件存入本地素材库。

### 1.3 触发时机

- **自动触发**: M-01（关键词扩展）完成后，由流水线引擎自动调用
- **人工触发**: 运营人员在素材管理后台手动输入关键词搜索并采集视频素材
- **定时触发**: 每日 06:00 UTC 执行热门关键词的视频素材预采集（配合 M-23 热点数据）

## 4. 核心处理逻辑

### 4.1 处理步骤

```
Step 1: 输入校验
  - 做什么: 校验 task_id 格式、topic_id 格式、search_queries 非空且不超过 30 个、platform 合法、collection_config 参数范围
  - 怎么判断: 使用 jsonschema 库校验，对比 2.1 中的 Schema
  - 输出: 校验通过 → 进入 Step 2; 校验失败 → 返回 status=failed，记录错误详情

Step 2: 加载偏好规则
  - 做什么: 如果输入包含 style_preference_id，从 M-25 偏好库加载视频和音频风格偏好
  - 怎么判断:
      - 调用 M-25 偏好查询接口，传入 style_preference_id
      - 若命中偏好（置信度 >= 70%），将偏好注入筛选参数
      - 视频偏好包含：偏好的场景类型（b-roll/航拍/特写）、转场风格偏好
      - 音频偏好包含：偏好的 BGM 风格、节奏范围（BPM 区间）、情绪类型
  - 输出: 筛选偏好对象 `style_filter_params`

Step 3: 视频 API 搜索
  - 做什么: 使用搜索关键词，调用 Pexels Video 和 Pixabay Video API 搜索视频
  - 怎么判断:
      - 对每个 search_query.keyword，分别调用 Pexels Video 和 Pixabay Video API
      - 每次请求最多获取 30 条结果
      - 按 relevance_score 排序请求优先级（高分关键词优先）
      - 记录每次 API 调用的响应状态和速率限制头
  - 输出: 原始视频搜索结果列表（URL、作者、时长、分辨率等元数据）

Step 4: 视频筛选过滤
  - 做什么: 对搜索结果进行多维度过滤
  - 怎么判断:
      - 时长过滤: min_duration <= duration <= max_duration
      - 分辨率过滤: width >= min_resolution.width 且 height >= min_resolution.height
      - 方向过滤: preferred_orientation='portrait' 时要求 height/width >= 1.2；preferred_orientation='landscape' 时要求 width/height >= 1.2
      - 清晰度评估: 下载视频前 3 秒片段进行清晰度分析（OpenCV 拉普拉斯方差）
      - 记录每项过滤原因和数量
  - 输出: 通过过滤的视频列表

Step 5: 视频内容分析
  - 做什么: 对通过过滤的视频进行内容语义分析
  - 怎么判断:
      - 提取视频关键帧（每秒 1 帧，最多 10 帧）
      - 调用 Gemini 视觉模型分析关键帧内容，生成 scene_type 和 content_tags
      - 将 content_tags 与搜索关键词进行语义相似度计算
      - 综合评分 = content_match * 0.4 + visual_quality * 0.3 + duration_suitability * 0.3
  - 输出: 每段视频的 scene_type、content_tags 和 relevance 评分

Step 6: 视频下载与保存
  - 做什么: 下载满足所有质量要求的视频到本地素材库
  - 怎么判断:
      - 按 overall 评分降序排列，下载前 max_clips_per_query 段视频
      - 文件命名遵循规范: `[选题编号]_[平台]_[日期]_video_[素材ID].ext`
      - 下载到 `data/files/materials/videos/YYYY/MM/` 目录
      - 生成视频封面缩略图（取第 1 秒关键帧，宽 400px）
      - 下载完成后校验文件完整性（文件大小 > 0，ffprobe 可正常解析）
      - 若视频文件损坏，删除并标记，尝试重新下载
  - 输出: 本地存储路径和视频元数据

Step 7: 音频 API 搜索
  - 做什么: 使用搜索关键词和情绪/节奏偏好，调用 Pixabay Music 和 Mixkit API 搜索 BGM
  - 怎么判断:
      - 使用 search_query.keyword 作为基础搜索词
      - 结合 mood_preferences 和 rhythm_preferences 构建搜索参数
      - 每次请求最多获取 20 条结果
  - 输出: 原始音频搜索结果列表

Step 8: 音频筛选与下载
  - 做什么: 筛选并下载满足要求的音频文件
  - 怎么判断:
      - 时长过滤: duration >= min_duration
      - 情绪匹配: API 返回的情绪标签与 mood_preferences 匹配
      - 版权检查: 若 copyright_flag=true，标记为需人工确认，不自动使用
      - 下载到 `data/files/materials/audio/YYYY/MM/` 目录
      - 生成音频波形图作为缩略图
      - 对下载的音频进行 BPM 检测（使用 librosa 库）
  - 输出: 音频存储路径和音频元数据（含情绪、节奏、BPM 标注）

Step 9: 写入数据库与返回
  - 做什么: 将视频和音频素材记录写入 `materials` 表和 `material_tags` 表
  - 怎么判断:
      - 为每个素材生成唯一 material_id（格式: MAT-YYYY-NNNNNN）
      - 视频素材: type='video', copyright_status='free'
      - 音频素材: type='audio', copyright_status='free'（若 copyright_flag=true 则设为 'unknown'）
      - 写入 `material_tags` 表（content_tags、mood、rhythm_type 等标签关联）
      - 写入 `module_outputs` 表完整输出 JSON
      - 返回最终的 Output JSON
  - 输出: 最终的 Output JSON
```

## 6. 错误处理策略

| 错误类型 | 处理方式 | 重试策略 | 告警级别 |
|---------|---------|---------|---------|
| 输入 JSON Schema 校验失败 | 返回 status=failed，错误详情写入 `errors` 表，终止执行 | 不重试 | ERROR |
| Pexels Video API 认证失败（401/403） | 跳过该数据源，仅使用 Pixabay Video；记录错误并告警 | 不重试 | CRITICAL |
| Pixabay Video API 认证失败（401/403） | 跳过该数据源，仅使用 Pexels Video；记录错误并告警 | 不重试 | CRITICAL |
| 视频 API 速率限制（429） | 等待 Retry-After 头指定的秒数后重试 | 3次，指数退避 2s/4s/8s | WARN |
| 音频 API 速率限制（429） | 等待 Retry-After 头指定的秒数后重试 | 3次，指数退避 2s/4s/8s | WARN |
| 视频搜索返回空结果 | 记录该关键词无结果，继续处理下一个关键词 | 不重试 | INFO |
| 音频搜索返回空结果 | 记录该关键词无结果，继续处理下一个关键词 | 不重试 | INFO |
| 视频下载超时（> 120s，视频文件较大） | 记录该视频为 failed，继续下载下一个 | 3次，指数退避 2s/4s/8s | WARN |
| 视频文件损坏（ffprobe 无法解析） | 删除损坏文件，重新下载；若仍损坏则丢弃并告警 | 重试 1 次 | ERROR |
| 音频文件损坏（无法播放） | 删除损坏文件，重新下载；若仍损坏则丢弃并告警 | 重试 1 次 | ERROR |
| BGM 版权标记（copyright_flag=true） | 不自动使用，标记 copyright_status='unknown'，通知人工确认 | 不重试 | WARN |
| 磁盘空间不足（写入失败） | 清理 `data/files/materials/tmp/` 临时目录，告警 | 清理后重试 1 次 | CRITICAL |
| LLM 视频内容分析异常（500/429/超时） | 降级：使用视频标题和 API 返回的描述作为 content_tags | 重试 1 次 | WARN |
| FFmpeg 关键帧提取失败 | 降级：使用视频第一帧（通过 OpenCV 截取）作为缩略图 | 不重试 | WARN |
| BPM 检测失败（音频分析异常） | bpm 字段设为 null，不影响整体流程 | 不重试 | INFO |
| 数据库写入失败（连接断开/锁超时） | 暂存结果到本地 JSON 文件，发送告警 | 5次，间隔 2s | CRITICAL |
| 全部视频和音频下载失败（status=failed） | 通知流水线引擎，写入 `pipeline_errors` 表 | 不重试 | CRITICAL |

### 降级策略

当模块完全不可用时（如所有 API 均不可达 + LLM 服务中断），启用以下降级方案：

1. **兜底视频库**: 使用 `data/files/materials/videos/default/` 中的预置默认视频片段（按场景类型分类存储，如科技类、自然类、城市类）
2. **兜底音频库**: 使用 `data/files/materials/audio/default/` 中的预置默认 BGM（按情绪/节奏分类存储）
3. **人工介入**: 发送企业微信/钉钉通知给运营团队，附上选题 ID 和关键词，由人工手动搜集视频和音频素材
4. **跳过本模块**: 在 `pipeline_config` 中设置 `video_collector.skip_on_failure = true`，流水线使用默认素材继续后续流程

|------|---------|

| P2 (系统主导) | 模块自动搜索、筛选、下载并入库。偏好库置信度 >= 70% 时使用偏好模板自动筛选。仅当 overall 评分 < 60 或 copyright_flag=true 时通知人工复核。 |
