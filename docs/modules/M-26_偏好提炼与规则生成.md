# M-26 偏好提炼与规则生成

---

## 一、模块定位

| 属性 | 值 |
|------|-----|
| 模块编号 | M-26 |
| 模块名称 | 偏好提炼与规则生成 |
| 所属分类 | 自学习 |
| 优先级 | P1 |
| 上游依赖 | M-25（学习数据收集）、M-28（初始偏好模板导入） |
| 下游依赖 | M-27（规则时效性管理）、M-29（学习成果应用） |
| 触发方式 | 定时任务（每4小时）+ M-25写入后回调 |

### 功能概述

本模块是自学习系统的**核心分析引擎**。从 M-25 收集的结构化学习记录中，通过频率统计、模式识别、正负强化等算法，提炼出用户的偏好规则，存入三大偏好库（素材偏好、风格偏好、策略偏好）。每条规则附带置信度评估，低于60%置信度的规则仅作参考，不强制执行。

### 设计目标

1. **从数据到规则**：将碎片化的审阅记录转化为可执行的偏好规则
2. **正负双向学习**：正面强化 + 负面规避双管齐下
3. **置信度驱动**：规则的可信度决定其执行力度
4. **三大偏好库**：素材、风格、策略三维度覆盖内容生产全流程

### 4.2 频率统计与规则优先级

```python
def frequency_analysis(records):
    """
    频率统计 → 高频变化 → 优先规则化
    """
    tag_frequency = {}
    
    for record in records:
        for tag in record.tags:
            key = f"{tag.tag_category}:{tag.tag_name}"
            if key not in tag_frequency:
                tag_frequency[key] = {
                    "count": 0,
                    "pass_count": 0,
                    "modify_count": 0,
                    "reject_count": 0,
                    "total_weight": 0
                }
            
            freq = tag_frequency[key]
            freq["count"] += 1
            freq["total_weight"] += tag.tag_weight
            
            if record.action_type == "pass":
                freq["pass_count"] += 1
            elif record.action_type == "modify":
                freq["modify_count"] += 1
            elif record.action_type == "reject":
                freq["reject_count"] += 1
    
    # 排序：优先处理高频标签
    sorted_tags = sorted(tag_frequency.items(), 
                         key=lambda x: x[1]["count"], 
                         reverse=True)
    
    # 仅处理频次 >= 3 的标签
    return {k: v for k, v in sorted_tags if v["count"] >= 3}
```

### 4.3 置信度计算

```python
def calculate_confidence(stats, total_records):
    """
    综合计算规则置信度
    
    考虑因素：
    1. 观察次数占比（样本量充足度）
    2. 正面/负面反馈比例
    3. 时间衰减（越近的数据权重越高）
    4. 标签权重累计
    """
    if total_records == 0:
        return 0
    
    # 1. 样本量因子（sigmoid函数，在10条时达到50%，20条时达到73%）
    sample_factor = 1 / (1 + math.exp(-0.3 * (stats["count"] - 10)))
    
    # 2. 一致性因子（正面操作占比）
    total_actions = stats["pass_count"] + stats["modify_count"] + stats["reject_count"]
    if total_actions > 0:
        consistency_factor = stats["pass_count"] / total_actions
    else:
        consistency_factor = 0.5
    
    # 3. 权重因子
    weight_factor = min(stats["total_weight"] / (stats["count"] * 1.0), 1.0)
    
    # 4. 综合置信度
    raw_confidence = (sample_factor * 0.3 + consistency_factor * 0.5 + weight_factor * 0.2) * 100
    
    # 5. 限制范围
    return round(min(max(raw_confidence, 0), 100), 1)
```

### 4.4 正负强化

| 操作类型 | 强化方向 | 权重变化 | 置信度变化 |
|---------|---------|---------|-----------|
| pass | 正面强化 | +0.05 | +2 |
| supplement | 正面强化 | +0.03 | +1 |
| modify (minor) | 轻微负面 | -0.02 | -1 |
| modify (moderate) | 中度负面 | -0.05 | -3 |
| modify (major) | 严重负面 | -0.10 | -5 |
| reject | 强负面 | -0.15 | -8 |

## 七、自学习

### 7.1 规则质量自评估

- 每生成10条新规则后，评估其对后续审阅通过率的实际影响
- 规则采纳率低于30%的自动降级
- 规则置信度连续3次更新下降的，标记为 "待确认"

### 7.2 提炼算法优化

## 九、Hermes Agent 指令

### 9.1 角色定义

```
你是 M-26 偏好提炼与规则生成模块的 Hermes Agent。
你的职责是：从 M-25 收集的学习记录中，通过统计分析和模式识别，
提炼出用户的偏好规则，存入三大偏好库，驱动后续内容生产的自动化优化。
```

### 9.2 执行指令

```
触发条件：
- 定时触发：每4小时执行一次
- 事件触发：M-25 写入新学习记录后回调

执行步骤：

1. 【数据拉取】从 learning_records 表读取数据：
   a. 筛选条件：processed_by_m26 = false
   b. 排序：按 created_at 升序
   c. 分批：每批最多500条
   d. 检查：若总记录数 < min_records_required（默认10），跳过本次处理

2. 【数据分组】将记录按维度分组：
   a. 第一层：按 category 分组（copy/cover/video/material/topic/strategy）
   b. 第二层：按 tags[].tag_category 分组（tone/color/duration/composition/...）
   c. 第三层：按 tags[].tag_name 分组（具体标签名）
   d. 过滤：每组记录数 < 3 的分组暂不处理

3. 【频率统计】对每个有效分组：
   a. 统计总出现次数 count
   b. 统计各 action_type 次数（pass/modify/reject/supplement）
   c. 统计累计权重 total_weight
   d. 排序：按 count 降序，优先处理高频标签

4. 【模式识别】检测复合模式：
   a. 同一 content_id 的多条记录：分析修改链
   b. 同一 reviewer_id 的偏好：提取审阅人特定倾向
   c. 同一 platform 的规律：识别平台特定偏好
   d. 时间维度模式：检测周期性偏好变化

5. 【规则生成/更新】对每个待处理的标签分组：
   a. 检查 preference_rules 中是否已有相似规则：
      - 相似度 > 80%：更新已有规则
      - 相似度 < 80%：创建新规则
   b. 新规则创建：
      - 生成 rule_id（PR-{8位hex}）
      - 确定 library（根据 tag_category 映射）
      - 确定 rule_category（根据 tag_name）
      - 计算初始 confidence
      - 设置 source 信息
      - 设置 status（根据 confidence）
   c. 已有规则更新：
      - 更新 statistics 计数
      - 重新计算 confidence
      - 更新 status 和 execution_mode
      - 递增 version

6. 【正负强化处理】：
   a. 对于 pass 操作的记录：
      - 找到对应规则，confidence += 2
      - statistics.positive_count += 1
   b. 对于 reject 操作的记录：
      - 找到对应规则，confidence -= 8
      - statistics.negative_count += 1
   c. 对于 modify 操作的记录：
      - 根据 severity 调整 confidence
      - minor: -1, moderate: -3, major: -5
   d. confidence 限制在 0-100 范围内

7. 【规则状态更新】根据最新 confidence 更新：
   a. confidence >= 90:
      - status = "active"
      - execution_mode = "default_execute"
   b. 60 <= confidence < 90:
      - status = "reference_only"
      - execution_mode = "suggest_reference"
   c. confidence < 60:
      - status = "pending_review"
      - execution_mode = null（不执行）
   d. confidence < 20 且 negative_count > positive_count * 3:
      - status = "deprecated"

8. 【数据持久化】：
   a. 批量 UPSERT preference_rules
   b. 批量 UPDATE learning_records SET processed_by_m26 = true, processed_at = NOW()
   c. 记录处理批次日志

9. 【结果汇总】返回处理摘要：
   {
     "batch_id": "BATCH-{timestamp}",
     "processed_at": "ISO datetime",
     "summary": {
       "total_records_processed": N,
       "new_rules_created": N,
       "existing_rules_updated": N,
       "rules_deprecated": N,
       "by_library": {
         "material": { "new": N, "updated": N },
         "style": { "new": N, "updated": N },
         "strategy": { "new": N, "updated": N }
       },
       "confidence_distribution": {
         "high_90plus": N,
         "medium_60_89": N,
         "low_below_60": N
       }
     },
     "warnings": [...]
   }
```

### 9.3 质量约束

```
- 规则生成准确率目标 >80%（生成规则在后续审阅中被采纳的比例）
- 置信度计算偏差 <5%（与人工评估对比）
- 单次处理延迟 <60秒
- 不允许在样本量 < 3 的情况下生成新规则
- 不允许 confidence 超出 0-100 范围
```
