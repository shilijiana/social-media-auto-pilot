# M-28 初始偏好模板导入

---

## 一、模块定位

| 属性 | 值 |
|------|-----|
| 模块编号 | M-28 |
| 模块名称 | 初始偏好模板导入 |
| 所属分类 | 自学习 |
| 优先级 | P3 |
| 上游依赖 | 无（首次运行触发） |
| 下游依赖 | M-26（偏好提炼与规则生成） |
| 触发方式 | 首次运行检测 + 手动导入 |

### 功能概述

本模块是自学习系统的**冷启动层**。在系统首次运行时，由于缺乏历史审阅数据，偏好库为空。本模块提供一个模板表单供人工填写初始偏好，包括封面风格、常用标签、话题方向、规避内容、平台优先级等，将这些初始偏好以80%置信度导入三大偏好库作为初始规则。后续随着实际审阅数据的积累，这些初始规则的置信度会根据验证结果自动调整。

### 设计目标

1. **冷启动友好**：无历史数据时也能建立初始偏好库
2. **结构化输入**：提供完整的模板表单，引导人工填写
3. **合理初始值**：80%初始置信度，既不过高也不过低
4. **可验证可调整**：初始规则可被后续数据验证或推翻

### 4.2 模板到规则映射

```python
def map_template_to_rules(template):
    """
    将偏好模板映射为三大偏好库的规则列表
    """
    rules = []
    initial_confidence = 80.0
    
    # === 素材偏好库映射 ===
    mp = template.get("material_preferences", {})
    cover = mp.get("preferred_cover_style", {})
    
    if cover.get("color_scheme"):
        cs = cover["color_scheme"]
        if cs.get("primary_colors"):
            rules.append(create_rule(
                library="material",
                rule_category="色调偏好",
                rule_name="封面主色调偏好",
                rule_content={
                    "preference_type": "positive",
                    "description": f"优先使用主色调: {', '.join(cs['primary_colors'])}",
                    "parameters": {"primary_colors": cs["primary_colors"]}
                },
                confidence=initial_confidence
            ))
        if cs.get("avoid_colors"):
            rules.append(create_rule(
                library="material",
                rule_category="色调偏好",
                rule_name="封面规避颜色",
                rule_content={
                    "preference_type": "negative",
                    "description": f"避免使用颜色: {', '.join(cs['avoid_colors'])}",
                    "parameters": {"avoid_colors": cs["avoid_colors"]}
                },
                confidence=initial_confidence
            ))
    
    if cover.get("composition"):
        comp = cover["composition"]
        rules.append(create_rule(
            library="material",
            rule_category="构图偏好",
            rule_name="封面构图风格",
            rule_content={
                "preference_type": "positive",
                "description": f"优先{comp['layout_style']}构图",
                "parameters": {"layout_style": comp["layout_style"]}
            },
            confidence=initial_confidence
        ))
    
    # BGM风格
    if mp.get("preferred_bgm_styles"):
        rules.append(create_rule(
            library="material",
            rule_category="音频偏好",
            rule_name="BGM风格偏好",
            rule_content={
                "preference_type": "positive",
                "description": f"优先BGM风格: {', '.join(mp['preferred_bgm_styles'])}",
                "parameters": {"bgm_styles": mp["preferred_bgm_styles"]}
            },
            confidence=initial_confidence
        ))
    
    # 视频时长
    if mp.get("preferred_video_duration"):
        dur = mp["preferred_video_duration"]
        rules.append(create_rule(
            library="material",
            rule_category="时长偏好",
            rule_name="视频时长偏好",
            rule_content={
                "preference_type": "positive",
                "description": f"短视频{dur['short_form']['min_seconds']}-{dur['short_form']['max_seconds']}秒",
                "parameters": {"duration_range": dur}
            },
            confidence=initial_confidence
        ))
    
    # 规避特征
    if mp.get("avoid_features"):
        rules.append(create_rule(
            library="material",
            rule_category="规避特征",
            rule_name="素材规避特征",

def create_rule(library, rule_category, rule_name, rule_content, confidence):
```

### 4.3 首次运行检测

```python
def check_first_run():
    """
    检测是否为首次运行
    """
    rule_count = db.count("preference_rules", "status != 'deprecated'")
    template_count = db.count("preference_templates")
    
    if rule_count == 0 and template_count == 0:
        return {"is_first_run": True, "message": "偏好库为空，建议导入初始偏好模板"}
    else:
        return {"is_first_run": False, "message": "偏好库已有数据，跳过模板导入"}
```

### 4.4 模板覆盖策略

- **first_run**: 无条件导入，创建所有规则
- **manual_import**: 检查规则是否已存在
  - 若 `override_existing=true`: 更新已有规则的 confidence 和 rule_content
  - 若 `override_existing=false`: 跳过已有规则，仅创建新规则
- **template_update**: 将旧模板标记为 superseded，导入新模板

## 六、错误处理

### 6.1 输入验证错误

| 错误码 | 场景 | 处理方式 |
|--------|------|---------|
| E2801 | 模板JSON格式无效 | 返回详细验证错误，指出具体字段 |
| E2802 | 必填字段缺失 | 返回缺失字段列表 |
| E2803 | 字段值超出允许范围 | 返回具体限制说明 |
| E2804 | 模板版本不兼容 | 提示升级模板版本 |

### 6.2 导入错误

| 错误码 | 场景 | 处理方式 |
|--------|------|---------|
| E2810 | 数据库写入失败 | 重试3次，回滚整个导入批次 |
| E2811 | 规则ID冲突 | 重新生成ID后重试 |
| E2812 | 模板已存在且状态为active | 根据 override_existing 决定 |
| E2813 | 批量导入部分失败 | 标记失败项，返回部分成功结果 |

### 6.3 业务错误

| 错误码 | 场景 | 处理方式 |
|--------|------|---------|
| E2820 | 非首次运行但无规则 | 异常状态，记录日志并允许导入 |
| E2821 | 模板映射规则数为0 | 提示用户至少填写一个偏好项 |
| E2822 | 初始置信度超出50-90范围 | 使用默认值80% |

## 七、自学习

### 7.1 模板质量评估

- 统计每个模板导入的规则后续验证通过率
- 通过率 <50% 的模板，标记为低质量
- 分析低质量模板的共同特征，优化模板表单设计

### 7.2 表单优化

- 收集用户填写模板的时间和修改次数
- 识别填写困难或频繁修改的字段
- 优化字段说明和默认值建议

### 7.3 初始置信度自适应

- 统计初始规则的平均存活时间
- 如果大部分初始规则在短时间内被推翻，降低默认置信度
- 如果大部分初始规则被验证正确，可适当提高默认置信度

## 九、Hermes Agent 指令

### 9.1 角色定义

```
你是 M-28 初始偏好模板导入模块的 Hermes Agent。
你的职责是：在系统首次运行时，引导用户填写偏好模板，
将人工输入的初始偏好转化为结构化的偏好规则，以80%置信度导入三大偏好库，
为自学习系统提供冷启动数据。
```

### 9.2 执行指令

```
触发条件：
- 首次运行检测（preference_rules 为空且 preference_templates 为空）
- 手动导入请求
- 模板更新请求

执行步骤：

1. 【首次运行检测】检查偏好库状态：
   a. 查询 preference_rules: COUNT WHERE status != 'deprecated'
   b. 查询 preference_templates: COUNT
   c. 若两者均为0 → 触发首次运行流程
   d. 若已有数据 → 根据请求类型决定（manual_import / template_update）

2. 【表单展示】向用户展示偏好模板表单：
   a. 分为三大区域：素材偏好、风格偏好、策略偏好
   b. 每个字段附带说明和示例
   c. 必填字段标注红色星号
   d. 提供"跳过"选项（允许部分字段不填）

3. 【表单验证】验证用户提交的模板：
   a. 检查 JSON 格式是否正确
   b. 检查必填字段是否已填写
   c. 检查字段值是否在允许范围内
   d. 检查颜色值格式（HEX #RRGGBB）
   e. 检查时间格式（HH:MM）
   f. 返回验证错误详情（如有）

4. 【模板解析】将表单数据映射为规则列表：
   a. 素材偏好 → material 库规则：
      - 封面配色 → "封面主色调偏好" + "封面规避颜色"
      - 字体风格 → "封面板式风格"
      - 构图偏好 → "封面构图风格"
      - BGM风格 → "BGM风格偏好"
      - 视频时长 → "视频时长偏好"
      - 规避特征 → "素材规避特征"
   b. 风格偏好 → style 库规则：
      - 标题模式 → "标题格式偏好"
      - 内容语气 → "内容语气偏好"
      - 标签习惯 → "话题标签使用习惯"
      - emoji使用 → "Emoji使用偏好"
      - 配色方案 → "品牌配色方案"
      - 字幕样式 → "字幕显示偏好"
      - 语速/节奏 → "视频节奏偏好"
   c. 策略偏好 → strategy 库规则：
      - 话题方向 → "偏好话题方向"
      - 规避类型 → "规避内容类型"
      - 平台优先级 → "平台优先级"
      - 内容格式 → "内容格式偏好"
      - 发布频率 → "发布频率与时间"

5. 【规则生成】对每条映射结果：
   a. 生成唯一 rule_id（PR-{8位hex}）
   b. 设置 library 和 rule_category
   c. 构建 rule_content（preference_type, description, parameters）
   d. 设置 confidence = 80（或用户指定的 initial_confidence）
   e. 设置 status = "reference_only"（80%置信度时）
   f. 设置 execution_mode = "suggest_reference"
   g. 设置 source.source_type = "template_import"
   h. 初始化 statistics（所有计数为0）

6. 【冲突检测】检查是否与已有规则冲突：
   a. 查询同 library + rule_category 的已有规则
   b. 若 override_existing=true → 更新已有规则
   c. 若 override_existing=false → 跳过，记录 skipped
   d. 若检测到直接矛盾（如已有规则为 positive 而新规则为 negative）
      → 记录警告，人工确认

7. 【数据持久化】写入数据库：
   a. 批量 INSERT preference_rules
   b. INSERT preference_templates（保存原始模板）
   c. 更新 preference_templates.created_rules
   d. 使用事务保证一致性

8. 【后续处理】导入完成后：
   a. 通知 M-26：初始规则已就绪，可开始偏好提炼
   b. 如果为 first_run，标记系统状态为 "initialized"
   c. 返回导入结果摘要

9. 【结果返回】返回导入结果：
   {
     "import_id": "IMP-{hex}",
     "imported_at": "ISO datetime",
     "template_id": "TPL-{hex}",
     "import_type": "first_run",
     "summary": {
       "total_rules_created": N,
       "by_library": {
         "material": N,
         "style": N,
         "strategy": N
       },
       "rules_skipped": N,
       "initial_confidence": 80,
       "warnings": [...]
     },
     "created_rules": [
       {
         "rule_id": "PR-{hex}",
         "library": "material",
         "rule_name": "封面主色调偏好",
         "confidence": 80,
         "status": "reference_only"
       },
       ...
     ]
   }
```

```
- 规则生成准确率 100%（映射逻辑不能出错）
```

### 9.4 自检指令

```
每次执行完成后，自检以下项目：
1. 检查模板中每个非空字段是否都生成了对应规则
2. 检查生成规则的 library 分类是否正确
3. 检查所有规则的 confidence 是否为设定值（默认80%）
4. 检查 preference_templates 记录是否写入成功
5. 检查规则数量是否合理（通常应在10-30条之间）
```

```
当收到 template_update 请求时：

2. 旧模板关联的规则不自动废弃（等待后续数据验证）
3. 导入新模板，创建新规则
4. 新旧规则之间可能存在冲突，由 M-26 在后续处理中解决
5. 在 preference_templates 中记录更新链（previous_template_id）
```
