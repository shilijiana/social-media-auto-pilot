```
M-10 位置说明:
  上游: M-07（素材匹配与调用模块）→ 传入视频/音频素材列表及存储路径
        M-08（文案差异化生成模块）→ 传入文案文本，用于 TTS 语音合成
  下游: M-11（内容精加工与定稿模块）→ 使用初剪视频进行精加工和人工审阅
  自学习: M-25/M-26 收集编辑偏好（时长、节奏、BGM、语速），优化剪辑参数库
```

### 1.2 核心职责

对原始视频/音频素材进行自动化初剪：将素材片段拼接、裁剪冗余部分、检测并移除静音和重复片段、基于 BGM 节奏进行节拍对齐剪辑。调用 TTS 服务生成 AI 配音并同步字幕。应用编辑偏好库（时长、节奏、BGM 类型、语速）中的用户偏好参数，确保视频风格与账号历史调性一致。按各平台视频导出预设（分辨率、码率、格式）输出初剪预览版。

### 1.3 触发时机

- **自动触发**: M-08（文案差异化生成模块）和 M-07（素材匹配与调用模块）均完成后，由流水线引擎自动调用
- **人工触发**: 运营人员在内容管理后台对某个内容草稿手动点击「生成视频初剪」或对已有视频点击「重新剪辑」
- **批量触发**: 支持一次性为多个内容草稿批量生成视频初剪

### 2.3 数据来源

| 字段 | 来源 |
|------|------|
| `task_id` | 流水线引擎生成 |
| `content_draft_id` | M-08（文案差异化生成模块）输出的 draft_id |
| `materials.video_clips` | M-07（素材匹配与调用模块）输出的 matched_materials 中 material_type='video' 的素材 |
| `materials.audio_tracks` | M-07 输出的 matched_materials 中 material_type='audio' 的素材 |
| `copywriting.platform_texts` | M-08 输出的各平台文案，经 TTS 文本预处理（去除 emoji、格式标记） |
| `target_platforms` | 分发计划配置，可运行时覆盖 |
| `editing_preferences` | M-26（偏好提炼与规则生成模块）管理的编辑偏好库，通过 `preference_api.get_editing(topic_tags, platforms)` 查询 |
| `generation_options` | 配置文件 `config/video_editing.yaml` 加载，支持运行时覆盖 |

## 4. 核心处理逻辑

### 4.1 处理步骤

```
Step 1: 输入校验与资源加载
  - 做什么: 校验输入格式，加载素材文件和编辑偏好
  - 怎么判断:
      - 使用 jsonschema 库校验输入格式
      - 校验所有 video_clips 文件路径可访问
      - 校验 audio_tracks 中至少有一个 bgm 类型的音轨（或有 copyright_clearance）
      - 校验 copywriting.platform_texts 中每个平台都有 tts_text
      - 如果 editing_preferences 缺失或 confidence < 0.7，调用 M-26 API 获取默认编辑偏好
      - 加载编辑偏好中的目标时长、BPM范围、TTS音色等参数
  - 输出: 素材文件列表（validated_materials）、编辑参数（editing_params）

Step 2: 素材预处理与拼接
  - 做什么: 对视频片段按 trim_in/trim_out 裁剪并拼接
  - 怎么判断:
      - 使用 FFmpeg 对每个片段按 trim_in/trim_out 进行精确裁剪
      - 按 relevance_score 降序排列片段（高相关度优先展示）
      - 使用 FFmpeg concat demuxer 拼接所有片段
      - 转场效果: 根据 transition_style 在片段间插入转场
        - cut: 硬切（无转场）
        - fade: 0.3s 淡入淡出
        - slide: 0.3s 滑动
        - zoom: 0.3s 缩放
        - none: 无转场
  - 输出: 拼接后的原始视频（concatenated_raw.mp4）

Step 3: 静音/重复片段检测与移除
  - 做什么: 分析音频轨道，检测并移除静音和重复片段
  - 怎么判断:
      - 使用 FFmpeg silencedetect 滤镜检测静音片段:
        - 参数: silence_threshold_db（默认 -40dB），min_silence_duration=0.5s
        - 标记静音区间（start_sec, end_sec）
      - 使用音频指纹（chromaprint）检测重复片段:
        - 将音频分段（每段 2s），计算音频指纹
        - 指纹相似度 > 90% 判定为重复
        - 保留第一个出现的片段，移除后续重复
      - 移除静音片段（保留非静音部分，最小片段 >= min_segment_duration_sec）
      - 移除重复片段（保留首次出现）
      - 使用 FFmpeg 重新拼接清理后的片段
  - 输出: 清理后的视频（cleaned.mp4）+ 静音/重复移除统计

Step 4: BGM 节拍检测与对齐
  - 做什么: 分析 BGM 节拍，将视频切点对齐到节拍上
  - 怎么判断:
      - 从 audio_tracks 中选择最匹配的 BGM:
        - 匹配条件: genre 匹配 preferred_genre，BPM 在 preferred_bpm_range 内
        - 多候选时选择 BPM 最接近范围中值的
        - 无匹配时选择默认 BGM（config/video_editing.yaml 中的 fallback_bgm）
      - 使用 librosa 或 aubio 进行节拍检测:
        - 提取 BGM 的 onset strength envelope
        - 使用动态阈值检测节拍点
        - 输出节拍时间点列表（beat_times）
      - 视频切点对齐:
        - 获取 cleaned.mp4 中每个片段的边界时间点
        - 将每个切点对齐到最近的节拍点（容差: beat_align_tolerance_ms）
        - 如果最近节拍点超出容差，保留原始切点
      - 使用 FFmpeg 按对齐后的切点重新切割和拼接
      - 将 BGM 作为音频轨道混入（音量比例: bgm_preferences.volume_ratio）
      - 原视频音轨保留（如果有），音量降为 30%
  - 输出: 节拍对齐后的视频（beat_synced.mp4）+ 切点映射表

Step 5: TTS 语音合成
  - 做什么: 为每个平台调用 TTS 服务生成 AI 配音
  - 怎么判断:
      - 对 target_platforms 中的每个平台:
        - 提取 copywriting.platform_texts[platform].tts_text
        - 调用 TTS API 进行语音合成:
          - voice_id: voice_preferences.tts_voice_id
          - speed: voice_preferences.tts_speed
          - pitch: voice_preferences.tts_pitch
          - format: mp3 或 wav
      - TTS 失败重试: 最多 max_tts_retries 次
      - 降级: 如果指定 voice_id 不可用，自动切换为备用音色
      - 返回 TTS 音频文件路径和字级时间戳（word_timestamps）
  - 输出: 每个平台的 TTS 音频文件 + 字级时间戳

Step 6: 配音与字幕同步
  - 做什么: 将 TTS 配音混入视频，生成同步字幕
  - 怎么判断:
      - 将 TTS 音频作为主音轨混入视频（音量: voice_volume_ratio）
      - 调整 BGM 音量在 TTS 播放时段自动降低（ducking，降至 15%）
      - 字幕生成:
        - 如果 enable_auto_subtitle=true:
          - 使用 TTS 返回的 word_timestamps 生成字幕
          - 按句子/短语分组（每组 10-15 个字符）
          - 生成 SRT 格式字幕文件
          - 使用 FFmpeg subtitles 滤镜将字幕烧录到视频
        - 字幕样式:
          - 字体: 默认中文字体
          - 字号: subtitle_font_size
          - 位置: 底部居中，距离底部 8% 画面高度
          - 样式: 白色文字 + 黑色描边/阴影（确保可读性）
  - 输出: 配音+字幕同步的视频（voiced_subtitled.mp4）+ SRT 字幕文件

Step 7: 时长适配
  - 做什么: 根据平台目标时长调整视频长度
  - 怎么判断:
      - 获取当前视频时长（current_duration）
      - 与 target_duration_sec[platform] 比较:
      - 如果 current_duration > target:
        - 加速策略: 对非关键片段进行 1.05x-1.15x 加速（保持音调）
        - 裁剪策略: 移除 relevance_score 最低的片段
        - 优先保持 TTS 配音完整性
      - 如果 current_duration < target:
        - 扩展策略: 对关键片段进行 0.9x-0.95x 慢放
        - 填充策略: 在开头/结尾添加转场或标题卡
        - 添加 B-roll（如果有额外素材）
      - 容差: 目标时长 ±5% 视为达标
  - 输出: 时长适配后的视频（duration_adjusted.mp4）

Step 8: 平台格式导出
  - 做什么: 按各平台视频规格导出最终视频
  - 怎么判断:
      - 平台导出预设:
        - douyin: 1080×1920 (9:16 竖屏), 30fps, H.264, 6000kbps, mp4
        - xiaohongshu: 1080×1440 (3:4 竖屏), 30fps, H.264, 5000kbps, mp4
        - bilibili: 1920×1080 (16:9 横屏), 30fps, H.264, 8000kbps, mp4
        - x: 1280×720 (16:9 横屏), 30fps, H.264, 4000kbps, mp4（或 1:1 正方形 1080×1080）
      - 如果源视频比例与目标比例不同:
        - 居中裁剪（crop）至目标比例
        - 缩放（scale）至目标分辨率
      - 使用 FFmpeg 进行编码导出
      - 同时生成低分辨率预览版（480p, 1000kbps）
      - 生成缩略图（取视频 10%/50%/90% 位置各一帧）
  - 输出: 各平台最终视频文件 + 预览版 + 缩略图

Step 9: 结果汇总与写入
  - 做什么: 汇总各平台结果，写入数据库
  - 怎么判断:
      - 为每个平台生成唯一 video_id（格式: VID-YYYY-NNNNNN）
      - 生成完整 timeline（包含每个片段的起止时间和来源素材）
      - 写入 video_drafts 表
      - 写入 tts_call_logs 表
      - 写入 module_outputs 表
      - 计算整体 status
  - 输出: 最终的 Output JSON
```

### 4.3 FFmpeg 关键命令示例

**素材拼接（concat demuxer）**:
```bash
# 生成文件列表
echo "file 'clip1_trimmed.mp4'" > concat_list.txt
echo "file 'clip2_trimmed.mp4'" >> concat_list.txt
# 拼接
ffmpeg -f concat -safe 0 -i concat_list.txt -c copy concatenated.mp4
```

**静音检测**:
```bash
ffmpeg -i input.mp4 -af "silencedetect=n=-40dB:d=0.5" -f null - 2>&1 | grep silencedetect
```

**TTS 配音混入 + BGM Ducking**:
```bash
ffmpeg -i video.mp4 -i tts_audio.mp3 -i bgm.mp3 \
  -filter_complex "[1:a]volume=1.0[tts];[2:a]volume=0.2[bgm];[bgm]asplit[bgm1][bgm2];[tts]asplit[tts1][tts2];[tts2]aformat=sample_rates=44100,sidechaincompress=threshold=0.1:ratio=4:level_sc=0.15[bgm_ducked];[bgm1][bgm_ducked]amix=inputs=2:duration=first[bgm_mix];[bgm_mix][tts1]amix=inputs=2:duration=first[aout]" \
  -map 0:v -map "[aout]" -c:v copy -c:a aac -b:a 192k output.mp4
```

## 6. 错误处理策略

| 错误类型 | 处理方式 | 重试策略 | 告警级别 |
|---------|---------|---------|---------|
| 输入 JSON Schema 校验失败 | 返回 status=failed，终止执行 | 不重试 | ERROR |
| 视频素材文件不存在/无法访问 | 跳过该片段，使用剩余素材继续。如果全部不可用，返回 failed | 不重试（该片段） | WARN/ERROR |
| 视频素材全部无音频轨道 | 跳过静音检测步骤，添加 warning: "素材无音频，跳过静音检测" | 不重试 | WARN |
| 无可用的 BGM 音轨（无 copyright_clearance） | 降级: 使用无版权默认 BGM（config 中配置的 fallback_bgm） | 不重试 | WARN |
| 静音检测无结果 | 跳过静音移除步骤，保留原始音频 | 不重试 | INFO |
| 节拍检测失败（BGM 无稳定节拍） | 跳过节拍对齐，使用原始切点，添加 warning | 不重试 | WARN |
| TTS API 返回 429 | 等待 Retry-After 后重试 | 3 次，间隔递增 | WARN |
| TTS API 返回 500/503 | 切换备用 TTS 服务（多服务商冗余） | 主服务 2 次 → 备用服务 2 次 | WARN |
| TTS 指定 voice_id 不可用 | 自动降级为同语言默认音色 | 不重试 | WARN |
| TTS 文本超长（> 服务商上限） | 自动分段（每段 500 字），逐段合成后拼接 | 不重试 | INFO |
| FFmpeg 执行超时（> 5 分钟） | 终止当前任务，记录错误，尝试降分辨率重试 | 1 次降分辨率 | WARN |
| FFmpeg 编码失败（不支持的编码参数） | 降级: 使用默认编码参数（H.264, 2000kbps）重试 | 1 次降级 | WARN |
| 磁盘空间不足（< 1GB） | 清理临时文件后重试。仍不足则终止并告警 | 1 次清理 | CRITICAL |
| 数据库写入失败 | 暂存结果到本地 JSON 文件 `data/m10_backup_{task_id}.json` | 5 次，间隔 2s | CRITICAL |
| 字幕生成失败 | 跳过字幕烧录，保留 SRT 文件单独提供 | 不重试 | WARN |
| 音频 fingerprint 库不可用 | 跳过重复检测步骤 | 不重试 | INFO |

### 降级策略

当核心服务不可用时：

1. **TTS 完全不可用**: 跳过配音，输出无配音版视频（仅 BGM + 画面），标记 `tts_status = "skipped"`。字幕仍生成但以硬字幕形式烧录
2. **FFmpeg 部分功能不可用**: 使用基础编码参数导出，标记 `encoding_quality = "degraded"`
3. **所有素材不可用**: 返回 status=failed，通知运营人员手动上传素材
4. **跳过本模块**: 在 `pipeline_config` 中设置 `video_editor.skip_on_failure = true`

### 7.3 阶段演进行为

|------|---------|
| P0 (人工主导) | 视频由剪辑师手动制作。模块仅负责将成品视频按平台规格转码导出。TTS 不启用。编辑偏好库为空。 |
| P1 (人机协同) | 模块自动生成初剪视频，剪辑师在后台审核和修改。TTS 启用。编辑偏好库开始积累数据。修改率高的参数（时长/BGM/语速）会触发偏好更新。 |
| P2 (系统主导) | 模块全自动生成视频，仅当完播率预测 < 50% 或人工设置审核标记时才通知人工。M-26 根据高完播率视频自动优化剪辑参数。支持 A/B 测试（不同节奏/BGM 变体，发布后自动选优）。 |
