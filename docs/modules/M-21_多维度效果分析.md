# M-21 多维度效果分析

---

## 1. 模块定位

| 属性 | 值 |
|------|-----|
| **模块编号** | M-21 |
| **模块名称** | 多维度效果分析 |
| **所属分类** | 数据分析 |
| **优先级** | P2（重要模块，分析链路核心） |
| **触发方式** | 定时任务（每日 10:00 UTC+8，在 M-20 之后执行）；支持手动触发 |
| **在系统中的位置** | 接收 M-20 清洗后的数据，进行横向对比、纵向对比和趋势分析，输出效果分析报告，供 M-24 下期计划草案生成使用 |

### 1.1 核心职责

1. **横向对比（跨平台）**：同一内容在 4 个平台（抖音/小红书/B站/微博）上的表现对比，识别各平台的优势与短板。
2. **纵向对比（历史 TOP10）**：当前内容与历史表现最佳的 TOP10 内容对比，提取爆款规律。
3. **趋势分析**：7 日 / 30 日关键指标变化曲线，判断账号整体走势（上升/平稳/下降）。
4. **效果分析报告**：综合以上三维分析，生成结构化的效果分析报告。

### 1.2 上下游关系

```
M-20(数据清洗与聚合) → M-21(本模块) → M-24(下期计划草案生成)
```

- **上游**：M-20 提供 `analytics_reports` 表中的清洗后数据
- **下游**：M-24 消费效果分析报告，用于生成下期内容计划草案

## 4. 核心逻辑

### 4.1 主流程

```
开始
  │
  ├─ 1. 加载数据
  │     ├─ 从 M-20 analytics_reports 读取指定周期的清洗后数据
  │     ├─ 加载跨平台内容映射表 (cross_platform_mapping)
  │     └─ 加载历史 TOP10 数据
  │
  ├─ 2. 横向对比分析（跨平台）
  │     ├─ 按 cross_platform_mapping 分组
  │     ├─ 对每组（同一内容多平台发布）：
  │     │   ├─ 提取各平台的 engagement_rate、views、likes、comments、shares
  │     │   ├─ 计算 platform_variance（各平台互动率方差）
  │     │   ├─ 识别 best_platform（互动率最高）
  │     │   ├─ 识别 worst_platform（互动率最低）
  │     │   └─ 生成 insights：
  │     │       - "内容 X 在抖音表现最佳（互动率 Y%），在小红书表现最差（互动率 Z%）"
  │     │       - "该内容的视频形式（抖音/B站）互动率明显高于图文形式（小红书/微博）"
  │     ├─ 汇总各平台优势：
  │     │   ├─ 按 platform 聚合 avg_engagement_rate
  │     │   ├─ 按 platform + content_type 交叉分析
  │     │   └─ 生成 strength_description 和 weakness_description
  │     └─ 汇总各平台短板
  │
  ├─ 3. 纵向对比分析（历史 TOP10）
  │     ├─ 加载历史 TOP10 内容（按 engagement_rate 降序取前 10）
  │     ├─ 提取爆款共性特征：
  │     │   ├─ 内容类型分布：short_video 占比？image_text 占比？
  │     │   ├─ 发布时间规律：周几？几点？分布热力图
  │     │   ├─ 话题标签统计：哪些话题高频出现在 TOP10 中？
  │     │   └─ 内容时长偏好：最优时长区间
  │     ├─ 生成 pattern_rules：
  │     │   - "短于 60 秒的视频互动率平均高出长视频 35%"
  │     │   - "周二至周四发布的互动率高于周末 20%"
  │     │   - "带「AI工具」标签的内容在 TOP10 中出现频率最高"
  │     └─ 对比当前周期表现：
  │         ├─ 当前周期 avg_engagement_rate vs 历史 avg
  │         ├─ 当前 TOP 内容 vs 历史 TOP10 最低门槛
  │         └─ 计算 improvement_pct
  │
  ├─ 4. 趋势分析
  │     ├─ 7 日趋势：
  │     │   ├─ 生成每日指标曲线数据（日期 → views, engagement_rate, followers）
  │     │   ├─ 线性回归计算斜率 slope
  │     │   ├─ IF slope > 0.05 → "rising"
  │     │   ├─ IF slope < -0.05 → "declining"
  │     │   ├─ IF std(engagement_rate) > 0.3 × mean → "volatile"
  │     │   └─ ELSE → "stable"
  │     ├─ 30 日趋势：
  │     │   ├─ 按周聚合数据
  │     │   ├─ 计算 momentum_score = (本周互动率 - 上周互动率) / 上周互动率
  │     │   └─ 判定趋势方向
  │     └─ 账号轨迹判定：
  │         ├─ 综合 7 日和 30 日趋势
  │         ├─ 判定 trajectory：
  │         │   - 连续上升 + momentum_score > 0.1 → "strong_growth"
  │         │   - 缓慢上升 → "steady_growth"
  │         │   - 基本持平 → "plateau"
  │         │   - 缓慢下降 → "early_decline"
  │         │   - 快速下降 → "steep_decline"
  │         └─ 基于历史趋势预测未来值
  │
  ├─ 5. 综合分析
  │     ├─ 合并三维分析结果
  │     ├─ 提取关键发现 key_findings
  │     ├─ 生成行动建议 action_items
  │     │   ├─ 根据平台短板 → 建议调整内容形式或发布时间
  │     │   ├─ 根据爆款规律 → 建议参考成功模式
  │     │   └─ 根据趋势 → 建议预警或加强
  │     └─ 按优先级排序（P1=趋势下降预警, P2=跨平台短板, P3=爆款规律应用）
  │
  └─ 6. 输出报告
        ├─ 写入 analytics_reports 表（分析章节）
        ├─ 生成效果分析报告文档
        └─ 通知 M-24 分析已完成
```

### 4.2 横向对比计算逻辑

```
Function cross_platform_analysis(cross_platform_groups, records):
    results = []

    FOR each group in cross_platform_groups:
        topic = group.content_topic
        title = group.content_title
        platform_performance = {}

        FOR platform, content_id in group.cross_platform_ids:
            record = find_record(records, content_id, platform)
            IF record found:
                platform_performance[platform] = {
                    "views": record.cleaned_metrics.views,
                    "engagement_rate": record.cleaned_metrics.engagement_rate,
                    "likes": record.cleaned_metrics.likes,
                    "comments": record.cleaned_metrics.comments,
                    "shares": record.cleaned_metrics.shares,
                    "performance_label": record.benchmark_comparison.performance_label
                }

        IF len(platform_performance) >= 2:
            # 计算平台间方差
            rates = [p["engagement_rate"] for p in platform_performance.values()]
            variance = sum((r - mean(rates))^2 for r in rates) / len(rates)

            best = max(platform_performance.items(), key=lambda x: x[1]["engagement_rate"])
            worst = min(platform_performance.items(), key=lambda x: x[1]["engagement_rate"])

            insights = generate_cross_platform_insights(platform_performance, best, worst, topic)

            results.append({
                "content_topic": topic,
                "content_title": title,
                "platform_performance": platform_performance,
                "best_platform": best[0],
                "worst_platform": worst[0],
                "platform_variance": variance,
                "insights": insights
            })

    RETURN results
```

```
Function extract_viral_patterns(historical_top10):

    day_distribution = count_by(publish_days)

    # 3. 话题分析
    all_topics = flatten([r.content_topic.split(",") for r in historical_top10])
    topic_frequency = count_by(all_topics)

    # 4. 内容时长分析
    duration_engagement = avg_by_group(historical_top10, "content_duration_bucket", "engagement_rate")

    # 5. 生成规律规则
    rules = []
    IF type_distribution["short_video"] >= 7:
```

### 4.4 趋势判定逻辑

```
Function judge_trajectory(daily_trends, weekly_trends):
    # 7 日线性回归斜率
    dates_7d = [i for i in range(7)]  # 标准化日期
    rates_7d = [t.avg_engagement_rate for t in daily_trends]
    slope_7d = linear_regression_slope(dates_7d, rates_7d)

    # 30 日动量
    weekly_rates = [w.avg_engagement_rate for w in weekly_trends]
    momentum_score = (weekly_rates[-1] - weekly_rates[0]) / weekly_rates[0]

    # 综合判定
    IF slope_7d > 0.05 AND momentum_score > 0.1:
        trajectory = "strong_growth"
        confidence = 0.9
    ELIF slope_7d > 0.01 AND momentum_score > 0.02:
        trajectory = "steady_growth"
        confidence = 0.8
    ELIF abs(slope_7d) <= 0.01 AND abs(momentum_score) <= 0.02:
        trajectory = "plateau"
        confidence = 0.85
    ELIF slope_7d < -0.01 AND momentum_score < -0.02:
        trajectory = "early_decline"
        confidence = 0.75
    ELIF slope_7d < -0.05 AND momentum_score < -0.1:
        trajectory = "steep_decline"
        confidence = 0.9
    ELSE:
        trajectory = "plateau"
        confidence = 0.5

    RETURN trajectory, confidence
```

## 7. 自学习机制

### 7.1 爆款规律迭代

```
每周分析结束后：
- 将本周 TOP3 内容加入历史 TOP10 候选池
- 如果新内容表现优于现有 TOP10 最低门槛 → 替换并更新 TOP10
- 重新提取 pattern_rules，计算新的置信度
- 标记置信度下降的规则 → 提示该规律可能失效
```

### 7.2 趋势预测模型校准

```
每月校准一次预测准确度：
- 对比上月预测值 vs 实际值
- 计算预测误差 MAPE（平均绝对百分比误差）
- IF MAPE > 20%：
  → 调整预测模型参数（如增加近期数据权重）
  → 增加外部因素（如平台算法更新、节假日）作为特征
```

### 7.3 跨平台洞察学习

```
运营人员可对 insights 进行"有用/无用"标注：
- 记录标注到 insights_feedback 表
- 累计同类 insight 被标记"无用" > 3 次 → 降低该类型洞察的生成优先级
- 累计同类 insight 被标记"有用" > 5 次 → 提升该类型洞察的生成优先级
```

```
你作为 M-21 多维度效果分析模块的 Hermes Agent，请按照以下步骤执行效果分析：

   - 加载 cross_platform_mapping 表（筛选有 2+ 平台映射的内容组）
   - 加载 historical_top10 表（全历史 TOP10 内容）
   - 加载近 30 天历史趋势数据

3. 横向对比分析（跨平台）：
   a. 按 cross_platform_mapping 分组
   b. 对每组：
      - 提取各平台的 engagement_rate、views、likes、comments、shares
      - 计算 platform_variance（各平台互动率的方差）
      - 找出 best_platform 和 worst_platform
      - 生成跨平台洞察：
        "内容「{title}」在 {best} 互动率最高（{rate}%），
         在 {worst} 最低（{rate}%），差异 {diff}%"
      - 如果某个平台互动率明显突出 → 分析可能原因（内容形式适配度）
   c. 汇总各平台优势与短板：
      - 计算各平台 avg_engagement_rate
      - 按 platform × content_type 交叉分析
      - 生成 platform_strengths 和 platform_weaknesses

4. 纵向对比分析（历史 TOP10）：
   a. 提取历史 TOP10 内容的共性特征：
      - 内容类型分布：统计各类型在 TOP10 中的频次
      - 发布时间规律：统计最优发布时间段和星期
      - 话题标签：统计高频话题
      - 内容时长：统计最优时长区间
   b. 生成 pattern_rules（爆款规律）：
      - 每条规则需包含置信度（频次/10）
      - 标记 applicable_platforms
   c. 当前周期 vs 历史对比：
      - 计算当前 avg_engagement_rate 与历史 avg 的差异
      - 如果当前有内容超过历史 TOP10 最低门槛 → 标记为潜在爆款

5. 趋势分析：
   a. 7 日趋势：
      - 生成每日指标曲线（views, engagement_rate, followers）
      - 计算线性回归斜率
      - 判定 trend_direction：rising / stable / declining / volatile
   b. 30 日趋势：
      - 按周聚合数据
      - 计算 momentum_score
      - 判定趋势方向
   c. 账号轨迹判定：
      - 综合 7 日和 30 日趋势
      - 判定 trajectory（strong_growth / steady_growth / plateau / early_decline / steep_decline）
      - 输出置信度
      - 预测下周和下月互动率

6. 综合分析：
   a. 合并三维分析结果
   b. 提取关键发现（key_findings），每条标记类别和严重程度
   c. 生成行动建议（action_items），按优先级排序：
      - P1: 趋势下降预警相关
      - P2: 跨平台短板改进
      - P3: 爆款规律应用
      - P4: 其他优化建议

7. 输出报告：
   - 写入 effect_analysis_reports 表
   - 更新 analytics_reports 表的 analysis 字段
   - 通知 M-24 分析已完成

注意事项：
- 数据不足时跳过对应分析维度并标记
- 方差为零时不做趋势判定
- 所有百分比保留 2 位小数
- insight 生成避免空泛描述，需有具体数据支撑
```
