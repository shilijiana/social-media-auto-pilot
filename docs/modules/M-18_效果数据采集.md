# M-18: 效果数据采集

## 1. 模块概述

| 属性 | 值 |
|------|-----|
| 模块编号 | M-18 |
| 模块名称 | 效果数据采集 |
| 所属类别 | 分发采集 |
| 优先级 | P0（最高优先级，核心模块） |
| 核心职责 | 在预设时间点（发布后 24 小时、72 小时）自动打开内容页面，采集数据指标：浏览量/展示量、点赞数、收藏/书签数、评论数、分享数、粉丝增长量，写入存储并标记时间戳+平台+内容ID，采集失败时标记重试 |
| 目标用户 | Hermes Agent（自动化采集）、数据分析师（数据消费） |

## 2. 输入输出定义

### 2.1 输入

| 输入项 | 来源模块 | 数据格式 | 触发条件 |
|--------|----------|----------|----------|
| 发布终态确认 | M-16/M-17 | `PublishFinalizedEvent` | 发布确认完成（published 终态） |
| 已发布内容URL | M-16 | `PublishTask.publish_url` | 发布成功后记录 |
| 采集时间调度 | 定时任务 | `CollectionSchedule` | 预设的 24h/72h 时间点 |
| 手动采集指令 | 前端/Hermes | `ManualCollectionCommand` | 用户手动触发数据采集 |

### 2.2 输出

| 输出项 | 目标 | 数据格式 | 说明 |
|--------|------|----------|------|
| 效果指标数据 | `performance_metrics` 表 | `PerformanceMetric` | 结构化的效果数据 |
| 采集异常标记 | 重试队列 | `CollectionFailure` | 采集失败时的重试标记 |
| 数据就绪通知 | M-20（数据分析） | `DataReadyEvent` | 通知下游数据已更新 |
| 异常波动告警 | 通知系统 | `AnomalyAlert` | 数据异常波动时告警 |

## 4. 核心业务逻辑

### 4.1 采集时间调度

```
内容发布成功
    │
    ├── T+24h ─── 第 1 次采集（24h 数据）
    │
    ├── T+72h ─── 第 2 次采集（72h 数据）
    │
    ├── T+7d  ─── 第 3 次采集（7 天数据，可选）
    │
    └── T+30d ─── 第 4 次采集（30 天数据，可选）
```

采集时间容差: ±15 分钟（避免精确整点被检测为机器行为）。

### 4.2 数据抓取流程

```
┌──────────────────┐
│  启动采集任务      │
└────────┬─────────┘
         ▼
┌──────────────────┐
│  打开内容页面      │  使用 Playwright 打开 publish_url
└────────┬─────────┘
         ▼
┌──────────────────┐
│  等待页面加载      │  等待关键指标元素出现
│  (最多30秒)       │
└────────┬─────────┘
         ▼
┌──────────────────┐
│  定位指标元素      │  平台特定 CSS 选择器
└────────┬─────────┘
         ▼
┌──────────────────┐
│  提取数值         │  解析文本 → 数字（处理"万"、K/M等缩写）
└────────┬─────────┘
         ▼
┌──────────────────┐
│  数据校验         │  范围检查、合理性检查
└────────┬─────────┘
         ▼
┌──────────────────┐
│  写入数据库       │  保存 PerformanceMetric 记录
└────────┬─────────┘
         ▼
┌──────────────────┐
│  异常检测         │  对比历史数据，标记异常
└────────┬─────────┘
         ▼
┌──────────────────┐
│  通知下游         │  发送 DataReadyEvent 给 M-20
└──────────────────┘
```

### 4.3 平台特定选择器配置

| 平台 | 浏览量 | 点赞 | 评论 | 分享 | 收藏 |
|------|--------|------|------|------|------|
| 抖音 | `.play-count` | `.like-count` | `.comment-count` | `.share-count` | `.collect-count` |
| 小红书 | `.note-view` | `.like-wrapper .count` | `.comment-input .count` | `.share-btn .count` | `.collect-btn .count` |
| B站 | `.video-data .view` | `.video-data .like` | `.video-data .reply` | `.video-data .share` | `.video-data .favorite` |
| Twitter | `[data-testid="app-text"]` (views) | `[data-testid="like"]` | `[data-testid="reply"]` | `[data-testid="retweet"]` | `[data-testid="bookmark"]` |

### 4.4 数值解析逻辑

```python
def parse_metric_value(raw_text: str) -> int:
    """解析平台返回的指标文本为整数"""
    text = raw_text.strip().lower().replace(",", "")
    
    # 处理中文单位
    if "万" in text:
        return int(float(text.replace("万", "")) * 10000)
    if "亿" in text:
        return int(float(text.replace("亿", "")) * 100000000)
    
    # 处理英文缩写
    if text.endswith("k"):
        return int(float(text[:-1]) * 1000)
    if text.endswith("m"):
        return int(float(text[:-1]) * 1000000)
    
    # 处理空值
    if text in ("", "-", "n/a", "--"):
        return None
    
    return int(text)
```

### 4.5 数据校验规则

| 校验项 | 规则 | 处理 |
|--------|------|------|
| 范围检查 | 浏览量 ≥ 0；互动数 ≤ 浏览量 | 标记异常，记录但不丢弃 |
| 一致性检查 | 24h 数据 ≤ 72h 数据 | 标记 `data_mismatch`，保留最新值 |
| 突增检查 | 单次采集值 > 前次值 × 10 | 标记 `sudden_spike`，记录但不丢弃 |
| 零互动检查 | 浏览量 > 100 但所有互动为 0 | 标记 `zero_engagement`，疑似降权 |
| 缺失检查 | 必填字段为空 | 标记 `is_complete=false`，记录缺失字段 |

### 4.6 采集失败重试策略

```
第 1 次失败 → 等待 10 分钟 → 重试
第 2 次失败 → 等待 1 小时 → 重试
第 3 次失败 → 等待 6 小时 → 重试
仍失败 → 标记 permanent_failure → 通知人工
```

## 6. 数据库表设计

### 6.1 performance_metrics 表

```sql
CREATE TABLE performance_metrics (
    metric_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    content_id UUID NOT NULL REFERENCES contents(content_id),
    publish_task_id UUID REFERENCES publish_tasks(task_id),
    platform VARCHAR(30) NOT NULL,
    content_url TEXT,
    collection_point VARCHAR(10) NOT NULL,
    metrics JSONB NOT NULL,
    data_quality JSONB DEFAULT '{"is_complete": true, "missing_fields": [], "estimated_fields": [], "confidence_score": 1.0}',
    collected_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    collection_duration_ms INTEGER,
    screenshot_url TEXT,
    raw_data JSONB,
    UNIQUE(content_id, platform, collection_point)
);

CREATE INDEX idx_pm_content ON performance_metrics(content_id);
CREATE INDEX idx_pm_platform ON performance_metrics(platform);
CREATE INDEX idx_pm_collected ON performance_metrics(collected_at);
```


## 7. 异常处理

| 异常场景 | 处理策略 |
|----------|----------|
| 页面加载超时（30秒） | 重试 1 次，仍超时则标记失败，进入重试队列 |
| 选择器失效（页面结构变更） | 截图保存 → 标记 `scraping_error` → 通知维护更新选择器 |
| 指标值为空 | 标记 `data_quality.is_complete=false`，记录缺失字段 |
| 反爬机制触发（验证码/拦截页） | 暂停该平台采集 1 小时 → 通知人工 |
| 并发采集冲突 | 同一 content+platform+point 的唯一约束保证幂等 |
| 数值解析异常 | 保存原始文本到 `raw_data`，标记 `estimated_fields` |

## 8. Hermes Agent 执行指令

```yaml
hermes_agent:
  module: M-18
  triggers:
    - event: publish.finalized
      source: M-17
      action: schedule_collections
    - schedule: "*/10 * * * *"
      action: process_collection_queue
    - command: manual_collect
      source: [user, hermes]
      action: execute_manual_collection
    - schedule: "0 */6 * * *"
      action: retry_failed_collections

  workflows:
    schedule_collections:
      description: 为已发布内容创建定时采集任务
      steps:
        - step: 计算采集时间点
          function: calculate_collection_schedule(publish_time)
          schedule:
            - point: "24h"
              scheduled_at: publish_time + 24h ± random(0, 900s)
            - point: "72h"
              scheduled_at: publish_time + 72h ± random(0, 900s)
            - point: "7d"
              scheduled_at: publish_time + 7d ± random(0, 1800s)
              optional: true
            - point: "30d"
              scheduled_at: publish_time + 30d ± random(0, 3600s)
              optional: true
        - step: 批量创建采集任务
          function: batch_insert_collection_tasks(tasks)
        - step: 确认创建结果
          function: respond({ scheduled_count: len(tasks), points: [t.collection_point for t in tasks] })

    process_collection_queue:
      schedule: every 10 minutes
      steps:
        - step: 查询到期采集任务
          function: get_due_collections()
          query: |
            SELECT * FROM collection_tasks
            WHERE status = 'scheduled'
            AND scheduled_at <= NOW() + INTERVAL '10 minutes'
          limit: 50
        - step: 并行执行采集
          loop: for task in due_tasks (max_concurrency=5)
          function: execute_collection(task.collection_id)

    execute_collection:
      description: 执行单次数据采集
      steps:
        - step: 更新任务状态
          function: update_collection_status(collection_id, "in_progress")
        - step: 打开浏览器页面
          function: open_content_page(content_url, platform)
          timeout: 30s
          on_error: handle_page_load_error
        - step: 等待指标元素加载
          function: wait_for_metric_elements(page, platform)
          selectors: platform_specific_selectors[platform]
          timeout: 15s
        - step: 提取各指标数据
          function: extract_all_metrics(page, platform)
          sub_steps:
            - extract_views()
            - extract_likes()
            - extract_comments()
            - extract_shares()
            - extract_saves()
            - extract_platform_specific()
        - step: 解析数值
          function: parse_metric_values(raw_values)
          parser: parse_metric_value()
        - step: 数据校验
          function: validate_metrics(parsed_values, content_id, platform)
          checks:
            - range_check()
            - consistency_check()
            - spike_detection()
            - zero_engagement_check()
        - step: 构建指标记录
          function: build_metric_record(content_id, platform, collection_point, parsed_values, validation_result)
        - step: 写入数据库
          function: insert_or_update_metric(metric_record)
          on_conflict: UPDATE (content_id, platform, collection_point)
        - step: 标记异常
          if: validation_result.has_anomalies
          function: insert_anomaly_flags(anomalies)
        - step: 截图保存
          function: take_screenshot(page, metric_id)
          optional: true
        - step: 更新任务状态
          function: update_collection_status(collection_id, "completed", completed_at=NOW())
        - step: 通知下游
          function: notify_data_ready(content_id, metric_id)

    handle_page_load_error:
      steps:
        - step: 判断错误类型
          function: classify_error(error)
        - step: 按类型处理
          switch: error_type
          cases:
            timeout:
              - function: retry_with_longer_timeout(collection_id, 60s)
            captcha:
              - function: pause_platform_collections(platform, duration=3600)
              - function: notify_admin("captcha_detected", { platform })
            blocked:
              - function: mark_collection_failed(collection_id, "blocked_by_platform")
              - function: notify_admin("access_blocked", { platform, url })
            not_found:
              - function: mark_collection_failed(collection_id, "content_not_found")
              - function: update_content_status(content_id, "可能已删除")

    retry_failed_collections:
      schedule: every 6 hours
      steps:
        - step: 查询失败且可重试的任务
          function: get_retryable_collections()
          query: |
            SELECT * FROM collection_tasks
            WHERE status = 'failed'
            AND retry_count < max_retries
            AND error_message NOT IN ('blocked_by_platform', 'content_not_found')
        - step: 逐个重试
          loop: for task in retryable_tasks
          function: retry_collection(task.collection_id)
          pre_action: increment_retry_count(collection_id)

    execute_manual_collection:
      description: 手动触发数据采集
      steps:
        - step: 验证内容存在且有发布URL
          function: validate_content_published(content_id, platform)
          on_error: abort("内容未发布或URL不存在")
        - step: 创建临时采集任务
          function: create_manual_collection_task(content_id, platform, collection_point="manual")
        - step: 立即执行采集
          function: execute_collection(collection_id)
        - step: 返回采集结果
          function: respond({ metric_id, metrics })

    extract_all_metrics:
      description: 提取页面所有指标数据
      platform_selectors:
        douyin:
          views: ".play-count"
          likes: ".like-count"
          comments: ".comment-count"
          shares: ".share-count"
          saves: ".collect-count"
          downloads: ".download-count"
          profile_visits: ".profile-visit-count"
        xiaohongshu:
          views: ".note-view"
          likes: ".like-wrapper .count"
          comments: ".comment-input .count"
          shares: ".share-btn .count"
          saves: ".collect-btn .count"
          interactions: ".interaction-count"
        bilibili:
          views: ".video-data .view"
          likes: ".video-data .like"
          comments: ".video-data .reply"
          shares: ".video-data .share"
          saves: ".video-data .favorite"
          coins: ".video-data .coin"
          danmaku: ".video-data .danmaku"
        twitter:
          views: '[data-testid="app-text"]'
          likes: '[data-testid="like"]'
          comments: '[data-testid="reply"]'
          shares: '[data-testid="retweet"]'
          saves: '[data-testid="bookmark"]'
          quote_tweets: '[data-testid="quote"]'
      fallback: |
        if primary_selector fails:
          try alternative_selectors[platform][metric]
          if all fail:
            mark_field_missing(metric)

    validate_metrics:
      validation_rules:
        range_check:
          - views >= 0
          - likes >= 0
          - comments >= 0
          - shares >= 0
          - saves >= 0
          - likes + comments + shares + saves <= views * 2
        consistency_check:
          - if previous_24h exists: 24h_values <= current_72h_values
        spike_detection:
          - if previous_metric exists: current_value <= previous_value * 10
        zero_engagement_check:
          - if views > 100 and all(likes, comments, shares, saves) == 0:
              flag "zero_engagement"
```
