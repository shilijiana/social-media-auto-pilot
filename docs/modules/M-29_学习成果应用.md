# M-29 学习成果应用

---

## 一、模块定位

| 属性 | 值 |
|------|-----|
| 模块编号 | M-29 |
| 模块名称 | 学习成果应用 |
| 所属分类 | 自学习 |
| 优先级 | P2 |
| 上游依赖 | M-26（偏好提炼与规则生成） |
| 下游依赖 | M-30（学习成效量化与报告） |
| 使用方模块 | M-02（文案生成）、M-03（封面生成）、M-08（视频剪辑）、M-09（素材筛选）、M-10（话题选择）、M-24（多平台适配） |
| 触发方式 | 被调用方模块在执行过程中主动调用 |

### 功能概述

本模块是自学习系统的**应用执行层**。在生产模块（M-02/M-03/M-08/M-09/M-10/M-24）执行内容生成任务时，自动调用偏好库查询接口，获取当前有效的偏好规则，将学习成果应用到内容生产中。同时记录规则的采纳情况和被修改情况，动态调整规则置信度，实现"学习→应用→反馈→再学习"的闭环。

### 设计目标

1. **无缝集成**：生产模块无需感知自学习细节，通过统一接口调用
2. **分级执行**：高置信度规则默认执行，低置信度规则建议参考
3. **闭环反馈**：记录规则采纳/修改情况，反向调整置信度
4. **可观测性**：记录每轮学习成果的应用效果

### 4.2 规则匹配算法

```python
def match_rules(query_request):
    """
    根据查询请求匹配适用的偏好规则
    """
    caller = query_request["caller_module"]
    content_type = query_request["content_type"]
    platform = query_request["target_platform"]
    topic_area = query_request.get("topic_area")
    libraries = query_request.get("query_options", {}).get("libraries")
    min_confidence = query_request.get("query_options", {}).get("min_confidence", 0)
    
    # 调用方到偏好库的映射
    caller_library_map = {
        "M-02": ["style"],           # 文案生成 → 风格偏好
        "M-03": ["material", "style"], # 封面生成 → 素材+风格偏好
        "M-08": ["material", "style"], # 视频剪辑 → 素材+风格偏好
        "M-09": ["material"],         # 素材筛选 → 素材偏好
        "M-10": ["strategy"],         # 话题选择 → 策略偏好
        "M-24": ["style", "strategy"] # 多平台适配 → 风格+策略偏好
    }
    
    target_libraries = libraries or caller_library_map.get(caller, ["material", "style", "strategy"])
    
    # 构建查询条件
    conditions = {
        "library": {"$in": target_libraries},
        "status": {"$in": ["active", "reference_only"]},
        "confidence": {"$gte": min_confidence}
    }
    
    # 平台过滤
    conditions["$or"] = [
        {"rule_content.constraints.platforms": {"$in": [platform]}},
        {"rule_content.constraints.platforms": {"$size": 0}},  # 无平台限制
        {"rule_content.constraints": None}  # 无约束条件
    ]
    
    # 内容类型过滤
    conditions["rule_content.constraints.content_types"] = {"$in": [content_type, None]}
    
    rules = db.query("preference_rules", conditions)
    
    # 计算优先级评分
    for rule in rules:
        rule["priority_score"] = calculate_priority(rule, query_request)
    
    # 排序：默认执行 > 建议参考，高优先级 > 低优先级
    rules.sort(key=lambda r: (
        0 if r["execution_mode"] == "default_execute" else 1,
        -r["priority_score"]
    ))
    
    return rules


def calculate_priority(rule, query_request):
    """
    计算规则优先级评分
    """
    score = rule["confidence"] * 0.6  # 置信度权重60%
    
    # 匹配度加分
    if rule["rule_content"].get("constraints"):
        constraints = rule["rule_content"]["constraints"]
        if query_request["target_platform"] in constraints.get("platforms", []):
            score += 15
        if query_request.get("topic_area") in constraints.get("topic_areas", []):
            score += 15
        if query_request["content_type"] in constraints.get("content_types", []):
            score += 10
    
    return min(score, 100)
```

### 4.3 置信度调整逻辑

```python
def process_feedback(feedback):
    """
    处理规则应用反馈，调整置信度
    """
    updates = []
    
    for applied in feedback["rules_applied"]:
        rule = get_rule(applied["rule_id"])
        if not rule:
            continue
        
        result = applied["application_result"]
        previous_confidence = rule["confidence"]
        new_confidence = previous_confidence
        
        if result == "adopted":
            # 采纳：置信度上升
            if rule["confidence"] < 90:
                new_confidence = min(rule["confidence"] + 3, 100)
            else:
                new_confidence = min(rule["confidence"] + 1, 100)
            
            # 更新统计
            rule["statistics"]["adoption_rate"] = recalculate_adoption_rate(rule)
            rule["statistics"]["last_verified_at"] = datetime.now().isoformat()
            
        elif result == "modified":
            modification = applied.get("modification_detail", {})
            mod_type = modification.get("modification_type", "parameter_adjustment")
            
            if mod_type == "parameter_adjustment":
                # 参数微调：小幅降级
                new_confidence = max(rule["confidence"] - 5, 0)
            elif mod_type == "partial_override":
                # 部分覆盖：中度降级
                new_confidence = max(rule["confidence"] - 10, 0)
            elif mod_type == "complete_override":
                # 完全覆盖：大幅降级
                new_confidence = max(rule["confidence"] - 20, 0)
            
            rule["statistics"]["override_count"] += 1
            rule["statistics"]["last_verified_at"] = datetime.now().isoformat()
            
        elif result == "rejected":
            # 拒绝：大幅降级
            new_confidence = max(rule["confidence"] - 15, 0)
            rule["statistics"]["override_count"] += 1
            
        elif result == "not_applicable":
            # 不适用：不调整置信度
            pass
        
        # 检查状态是否需要变更
        status_change = check_status_change(rule, previous_confidence, new_confidence)
        
        rule["confidence"] = round(new_confidence, 1)
        rule["updated_at"] = datetime.now().isoformat()
        
        updates.append({
            "rule_id": rule["rule_id"],
            "previous_confidence": previous_confidence,
            "new_confidence": rule["confidence"],
            "change_reason": f"应用结果: {result}",
            "status_change": status_change
        })
        
        # 持久化更新
        db.update("preference_rules", rule["rule_id"], rule)
    
    return updates


def check_status_change(rule, prev_conf, new_conf):
    """
    检查规则状态是否需要变更
    """
    change = {
        "previous_status": rule["status"],
        "new_status": rule["status"],
        "previous_execution_mode": rule["execution_mode"],
        "new_execution_mode": rule["execution_mode"]
    }
    
    # 置信度 ≥ 90%：升级为默认执行
    if prev_conf < 90 and new_conf >= 90:
        change["new_status"] = "active"
        change["new_execution_mode"] = "default_execute"
    
    # 置信度从 ≥90% 降到 <90%：降级为建议参考
    elif prev_conf >= 90 and new_conf < 90:
        change["new_status"] = "reference_only"
        change["new_execution_mode"] = "suggest_reference"
    
    # 置信度 < 60%：标记为待审核
    elif new_conf < 60:
        change["new_status"] = "pending_review"
        change["new_execution_mode"] = None
    
    return change if change["previous_status"] != change["new_status"] else None
```

### 4.4 规则冲突检测

```python
def detect_conflicts(matched_rules):
    """
    检测匹配规则之间的冲突
    """
    conflicts = []
    
    # 按 rule_category 分组
    by_category = {}
    for rule in matched_rules:
        cat = rule["rule_category"]
        if cat not in by_category:
            by_category[cat] = []
        by_category[cat].append(rule)
    
    for category, rules in by_category.items():
        if len(rules) > 1:
            # 检查是否有 positive 和 negative 同时存在
            positives = [r for r in rules if r["rule_content"]["preference_type"] == "positive"]
            negatives = [r for r in rules if r["rule_content"]["preference_type"] == "negative"]
            
            if positives and negatives:
                conflicts.append({
                    "rule_ids": [r["rule_id"] for r in positives + negatives],
                    "conflict_description": f"类别'{category}'下同时存在正面和负面偏好规则",
                    "resolution": "优先采用置信度较高的规则，人工审核后决定"
                })
    
    return conflicts
```

## 七、自学习

### 7.1 采纳率趋势分析

- 统计各调用方模块的规则采纳率变化趋势
- 采纳率持续下降 → 偏好库可能过时，触发 M-26 重新提炼
- 采纳率持续上升 → 偏好库质量提升，考虑提高自动执行比例

### 7.2 规则价值评估

- 统计每条规则被查询和采纳的次数
- 低频查询规则（<5次/月）标记为低价值
- 高频查询低采纳规则 → 分析原因，可能需要调整或废弃

### 7.3 置信度调整策略优化

- 分析 adopted/modified/rejected 的置信度调整效果
- 如果某类调整导致规则频繁在 active 和 reference_only 间切换 → 调整幅度过大
- 自适应调整步长

```sql
-- 查询各调用方模块的采纳率
SELECT 
  caller_module,
  COUNT(*) as total_applications,
  SUM(CASE WHEN application_result = 'adopted' THEN 1 ELSE 0 END) as adopted,
```

## 九、Hermes Agent 指令

### 9.1 角色定义

```
你是 M-29 学习成果应用模块的 Hermes Agent。
你的职责是：作为自学习系统与生产模块之间的桥梁，
为生产模块提供偏好规则查询服务，收集规则应用反馈，
动态调整规则置信度，实现"学习→应用→反馈→再学习"的闭环。
```

### 9.2 执行指令

```
【查询模式】当收到生产模块的偏好查询请求时：

1. 【请求验证】验证调用方合法性：
   a. 检查 caller_module 是否在允许列表中（M-02/M-03/M-08/M-09/M-10/M-24）
   b. 检查 content_type 和 target_platform 是否有效
   c. 记录查询日志（query_id, caller_module, timestamp）

2. 【规则匹配】从 preference_rules 表中匹配适用规则：
   a. 确定目标偏好库：
      - M-02 → [style]
      - M-03 → [material, style]
      - M-08 → [material, style]
      - M-09 → [material]
      - M-10 → [strategy]
      - M-24 → [style, strategy]
   b. 过滤条件：
      - status IN ('active', 'reference_only')
      - library IN target_libraries
      - confidence >= min_confidence（默认0）
      - 平台匹配（constraints.platforms 包含目标平台或无限制）
      - 内容类型匹配（constraints.content_types 包含目标类型或无限制）
   c. 话题领域匹配（可选）：constraints.topic_areas 包含目标话题或无限制

3. 【规则排序】按以下优先级排序：
   a. execution_mode: "default_execute" 优先于 "suggest_reference"
   b. priority_score 降序（综合考虑置信度和匹配度）
   c. confidence 降序（同优先级时）

4. 【冲突检测】检查匹配规则之间是否存在冲突：
   a. 按 rule_category 分组
   b. 检查同一类别下是否同时存在 positive 和 negative 规则
   c. 如有冲突，在 instructions.conflict_warnings 中说明

5. 【结果组装】构建查询结果：
   a. 分离 mandatory_rules（execution_mode=default_execute）
   b. 分离 suggested_rules（execution_mode=suggest_reference）
   c. 生成执行指令（instructions）
   d. 生成摘要统计（summary）

6. 【结果返回】返回格式：
   {
     "query_id": "PQ-{hex}",
     "queried_at": "ISO datetime",
     "rules": [...],
     "summary": { ... },
     "instructions": {
       "mandatory_rules": ["PR-xxx", ...],
       "suggested_rules": ["PR-yyy", ...],
       "conflict_warnings": [...]
     }
   }

7. 【缓存策略】：
   a. 缓存查询结果（TTL: 5分钟）
   b. 缓存键：{caller_module}:{content_type}:{platform}:{topic_area}
   c. 如果 preference_rules 有更新，主动失效相关缓存


【反馈模式】当收到生产模块的规则应用反馈时：

1. 【反馈验证】验证反馈数据：
   a. 检查 content_id 和 production_round 是否有效
   b. 检查 rules_applied 中每个 rule_id 是否存在
   c. 检查 application_result 是否为有效枚举值

2. 【逐条处理】对每个应用的规则：
   a. 获取规则当前状态（confidence, status, statistics）
   b. 根据 application_result 调整置信度：
      - "adopted":
        * 当前置信度 < 90: confidence += 3
        * 当前置信度 ≥ 90: confidence += 1
        * 更新 adoption_rate
        * 更新 last_verified_at
      - "modified":
        * parameter_adjustment: confidence -= 5
        * partial_override: confidence -= 10
        * complete_override: confidence -= 20
        * override_count += 1
      - "rejected":
        * confidence -= 15
        * override_count += 1
      - "not_applicable":
        * 不调整置信度
   c. 置信度限制在 0-100 范围内

3. 【状态升级/降级】检查规则状态是否需要变更：
   a. confidence ≥ 90 且之前 < 90:
      - status → "active"
      - execution_mode → "default_execute"
      - 记录：规则从"建议参考"升级为"默认执行"
   b. confidence < 90 且之前 ≥ 90:
      - status → "reference_only"
      - execution_mode → "suggest_reference"
      - 记录：规则从"默认执行"降级为"建议参考"
      - 原因：被手动修改，需重新验证
   c. confidence < 60:
      - status → "pending_review"
      - execution_mode → null
      - 记录：规则置信度过低，暂停应用

4. 【特殊处理】默认执行规则被手动修改时：
   a. 这是重要信号：用户主动覆盖了高置信度规则
   b. confidence 额外再降 10（总计降 15-30）
   c. 记录 modification_detail 供 M-26 分析
   d. 触发 M-26 重新评估相关规则

5. 【指标计算】计算本轮和累计采纳率：
   a. adoption_rate = adopted / total_applied
   b. modification_rate = modified / total_applied
   c. 对比历史数据，判断趋势（improving/stable/declining）

6. 【持久化】写入数据库：
   a. 更新 preference_rules（confidence, status, statistics）
   b. 写入 rule_application_feedback 表
   c. 写入 confidence_change_log 表

7. 【结果返回】返回反馈处理结果：
   {
     "feedback_id": "FB-{hex}",
     "processed_at": "ISO datetime",
     "summary": {
       "total_rules_applied": N,
       "adopted_count": N,
       "modified_count": N,
       "rejected_count": N
     },
     "confidence_updates": [...],
     "adoption_metrics": { ... }
   }
```

### 9.3 质量约束

```
- 查询响应时间 < 200ms（含缓存命中场景 < 50ms）
- 反馈处理响应时间 < 500ms
- 规则匹配准确率 100%
- 置信度调整精度 ±1
- 缓存一致性：规则更新后5秒内缓存失效
- 不允许返回 status=deprecated 或 status=pending_review 的规则
- 不允许返回 confidence < 0 的规则
```

### 9.4 自检指令

```
每次查询完成后，自检：
1. 检查返回规则中是否有 deprecated 或 pending_review 状态的规则
2. 检查 mandatory_rules 中的规则是否确实 confidence ≥ 90
3. 检查 conflict_warnings 是否正确识别

每次反馈处理完成后，自检：
1. 检查置信度更新是否在 0-100 范围内
2. 检查是否有规则从 default_execute 被降级（应伴随 override 记录）
3. 检查采纳率计算是否正确
4. 对比处理前后的规则数量和状态分布
5. 如发现异常，记录详细日志并通知 M-30
```
