# M-16: 多平台分发

## 1. 模块概述

| 属性 | 值 |
|------|-----|
| 模块编号 | M-16 |
| 模块名称 | 多平台分发 |
| 所属类别 | 分发采集 |
| 优先级 | P0（最高优先级，核心模块） |
| 核心职责 | 浏览器环境初始化（启动实例、加载用户画像、随机化窗口指纹），登录流程（检测登录状态→自动填充密码→验证码识别→等待人工干预），平台专属发布逻辑（抖音/小红书/B站/Twitter），账号故障转移（被封/受限自动切换备用账号），记录每次发布状态 |
| 目标用户 | Hermes Agent（自动化执行）、运营人员（人工干预） |

## 2. 输入输出定义

### 2.1 输入

| 输入项 | 来源模块 | 数据格式 | 触发条件 |
|--------|----------|----------|----------|
| 终稿内容 | M-11（内容编辑） | `FinalContent` | 内容通过审阅，标记为"就绪发布" |
| 平台账号凭证 | 系统配置/密钥管理 | `PlatformCredentials` | 发布任务创建时读取 |
| 发布调度指令 | 定时任务/Hermes | `PublishSchedule` | 按预定时间触发发布 |
| 人工干预响应 | 前端/通知系统 | `HumanInterventionResponse` | 登录验证码等需要人工介入时 |

### 2.2 输出

| 输出项 | 目标 | 数据格式 | 说明 |
|--------|------|----------|------|
| 发布任务记录 | `publish_tasks` 表 | `PublishTask` | 每次发布的任务记录 |
| 发布日志 | `publish_logs` 表 | `PublishLog` | 发布过程的详细日志 |
| 发布状态更新 | M-17（发布监控） | `PublishStatusUpdate` | 触发 M-17 开始监控 |
| 异常告警 | 通知系统 | `Alert` | 发布失败等关键告警 |

## 4. 核心业务逻辑

### 4.1 发布主流程

```
┌──────────────┐
│  初始化浏览器   │  启动 Playwright 实例，加载随机窗口指纹
└──────┬───────┘
       ▼
┌──────────────┐
│  加载用户画像   │  加载 Cookie、LocalStorage、浏览器扩展
└──────┬───────┘
       ▼
┌──────────────┐     ┌──────────────────────────────┐
│  检测登录状态   │────▶│ 未登录 → 自动填充 → 验证码识别    │
└──────┬───────┘     │          ↓                    │
       │             │    验证码出现 → 等待人工干预      │
       │             │          ↓                    │
       │             │    登录成功 → 持久化 Cookie     │
       │             └──────────────────────────────┘
       ▼
┌──────────────┐
│  导航至发布页   │  平台特定 URL 路由
└──────┬───────┘
       ▼
┌──────────────┐
│  填充内容      │  标题、正文、标签（平台特定适配）
└──────┬───────┘
       ▼
┌──────────────┐
│  上传媒体      │  图片/视频上传，等待转码完成
└──────┬───────┘
       ▼
┌──────────────┐
│  提交发布      │  点击发布按钮，等待响应
└──────┬───────┘
       ▼
┌──────────────┐
│  记录结果      │  保存发布 URL、截图、日志
└──────────────┘
```

### 4.2 浏览器环境初始化

1. **启动 Playwright 实例**: 使用 `playwright.chromium.launchPersistentContext()` 加载独立用户目录
2. **随机化窗口指纹**:
   - 随机 Viewport 尺寸（常见分辨率池中选取）
   - 随机 User-Agent（与平台匹配的现代浏览器 UA）
   - 随机 WebGL 指纹参数
   - 随机时区和语言设置
3. **加载反检测插件**: 注入 stealth 脚本，隐藏自动化特征
4. **代理配置**: 如果配置了代理，通过 `--proxy-server` 参数设置

### 4.3 登录流程

| 平台 | 登录URL | 特殊处理 |
|------|---------|----------|
| 抖音 | `creator.douyin.com` | 支持手机验证码登录，二维码登录 |
| 小红书 | `creator.xiaohongshu.com` | 手机验证码登录为主 |
| B站 | `member.bilibili.com` | 账号密码 + 极验验证码 |
| Twitter | `twitter.com/login` | 账号密码 + 可能的 Arkose 验证码 |

**验证码处理策略**:
1. 检测验证码类型（滑块、点选、文字识别）
2. 尝试自动识别（接入验证码识别服务）
3. 自动识别失败 → 截图保存 → 发送人工干预通知 → 等待最多 30 分钟
4. 超时 → 标记任务失败，安排重试

### 4.4 平台特定发布逻辑

#### 抖音发布
- 视频必须经过平台转码（等待转码完成）
- 标题限制 55 字
- 支持 @ 提及和话题标签
- 可选：定时发布、位置标记、商品橱窗

#### 小红书发布
- 图片最多 18 张
- 标题限制 20 字
- 正文限制 1000 字
- 支持话题标签和地点
- 笔记类型：图文/视频

#### B站发布
- 视频需先上传至创作中心
- 支持分P投稿
- 封面需单独上传（或从视频截取）
- 标签最多 10 个
- 支持定时发布

#### Twitter 发布
- 文本限制 280 字符（Blue 用户 25000）
- 图片最多 4 张
- 视频最长 140 秒
- 支持投票和引用推文

### 4.5 账号故障转移

```
主账号发布 → 检测到封禁/受限
    │
    ▼
查询备用账号（同平台、status=active）
    │
    ├── 有可用备用账号 → 切换账号 → 重新发布 → 标记 task.failed_over=true
    │
    └── 无可用备用账号 → 标记任务失败 → 发送告警 → 通知人工处理
```

故障转移规则:
- 每个任务最多故障转移 1 次
- 故障转移后原主账号标记为 `banned/restricted`
- 故障转移后的任务 `backup_account_id` 字段记录备用账号

### 4.6 串行发布策略

**CRITICAL**: 多平台发布必须串行执行，平台间加入随机间隔。

```
平台1发布 → 等待随机间隔(3-10分钟) → 平台2发布 → 等待随机间隔(3-10分钟) → ...
```

- 随机间隔: 180-600 秒之间的均匀分布随机值
- 串行原因: 避免同一内容多平台同时发布被检测为机器行为
- 发布顺序: 按平台优先级（可配置）

## 7. 异常处理

| 异常场景 | 处理策略 |
|----------|----------|
| 浏览器启动失败 | 重试 3 次，间隔 10 秒，仍失败则标记任务失败 |
| 登录 Cookie 过期 | 自动触发登录流程，使用存储的密码填充 |
| 验证码连续失败 3 次 | 暂停该账号 2 小时，通知人工处理 |
| 媒体上传超时（>10分钟） | 重试 1 次，仍超时则跳过该平台，标记部分失败 |
| 发布页面结构变更 | 检测选择器失效 → 截图保存 → 标记任务失败 → 通知维护 |
| 账号被封/受限 | 触发故障转移 → 切换到备用账号 |
| 网络中断 | 记录当前步骤 → 等待网络恢复 → 从断点继续 |
| 平台限流（429） | 退避重试：等待 5 分钟 → 10 分钟 → 30 分钟 |

## 8. Hermes Agent 执行指令

```yaml
hermes_agent:
  module: M-16
  triggers:
    - event: content.ready_for_publish
      source: M-11
      action: create_publish_tasks
    - schedule: "*/5 * * * *"
      action: process_pending_tasks
    - command: start_publish
      source: [hermes, user]
      action: execute_publish
    - event: captcha.answered
      source: user
      action: resume_publish_after_captcha

  workflows:
    create_publish_tasks:
      description: 为就绪内容创建多平台发布任务
      steps:
        - step: 解析目标平台列表
          function: resolve_target_platforms(content.target_platforms)
        - step: 对每个平台创建发布任务
          loop: for platform in target_platforms
          function: create_single_task(content_id, platform)
          sub_steps:
            - step: 内容平台适配
              function: adapt_content_for_platform(content, platform)
            - step: 选择主账号
              function: select_primary_account(platform)
              fail_if: no active primary account
              fallback: select_backup_account(platform)
            - step: 计算发布时间
              function: calculate_publish_time(platform, preferred_windows)
            - step: 插入发布任务
              function: insert_publish_task(task_data)
        - step: 通知 M-17 注册监控
          function: register_monitoring(task_ids)

    process_pending_tasks:
      schedule: every 5 minutes
      steps:
        - step: 查询到期待发布任务
          function: get_due_tasks()
          query: SELECT * FROM publish_tasks WHERE status='pending' AND scheduled_at <= NOW()
        - step: 串行执行发布
          loop: for task in due_tasks (ordered by platform_priority)
          function: execute_publish(task.task_id)
          post_action: wait_random_interval(180, 600)

    execute_publish:
      description: 执行单个平台的发布流程
      steps:
        - step: 更新任务状态
          function: update_task_status(task_id, "browser_init")
        - step: 初始化浏览器
          function: launch_browser(platform, account_id)
          details:
            - launchPersistentContext(user_profile_path)
            - apply_random_fingerprint()
            - inject_stealth_scripts()
          timeout: 60s
          on_error: retry(3, interval=10s) >> mark_failed
        - step: 加载用户画像
          function: load_user_profile(context, account_id)
          details:
            - restore_cookies(cookie_json)
            - restore_local_storage()
          status: "profile_load"
        - step: 检测登录状态
          function: check_login_state(page, platform)
          status: "login_check"
          if: not logged_in
          then: execute_login_flow
        - step: 导航到发布页
          function: navigate_to_publish_page(page, platform)
          status: "navigate_publish"
          wait_for: publish_form_selector
          timeout: 30s
        - step: 填充发布内容
          function: fill_publish_form(page, publish_content, platform)
          status: "filling_content"
          sub_steps:
            - fill_title()
            - fill_body()
            - fill_hashtags()
            - fill_platform_specific_fields()
        - step: 上传媒体文件
          function: upload_media_files(page, media_files, platform)
          status: "uploading_media"
          timeout: 600s
          wait_for: upload_complete_indicator
        - step: 提交发布
          function: submit_publish(page, platform)
          status: "submitting"
          wait_for: success_indicator OR error_indicator
          timeout: 60s
        - step: 验证发布结果
          function: verify_publish_result(page, platform)
          screenshot: true
        - step: 记录结果
          function: record_publish_result(task_id, result)
          details:
            - update_task_status("published")
            - set publish_url
            - set completed_at
        - step: 通知 M-17
          function: notify_publish_completed(task_id, publish_url)

    execute_login_flow:
      steps:
        - step: 导航到登录页
          function: navigate_to_login(page, platform)
          status: "login_execute"
        - step: 填充账号密码
          function: fill_credentials(page, username, password_decrypted)
        - step: 检测验证码
          function: detect_captcha(page)
          if: captcha_detected
          then:
            - function: attempt_auto_solve(captcha_type)
              timeout: 30s
              if: auto_solve_failed
              then:
                - function: screenshot_captcha()
                - function: send_human_intervention_request(task_id, captcha_screenshot)
                - function: update_task_status("waiting_human")
                - action: suspend_workflow
        - step: 提交登录
          function: submit_login(page)
          wait_for: logged_in_indicator
          timeout: 30s
        - step: 持久化 Cookie
          function: save_cookies(context, account_id)

    handle_account_failover:
      trigger: account_banned_or_restricted
      steps:
        - step: 更新主账号状态
          function: update_account_status(account_id, "banned")
        - step: 查找备用账号
          function: find_backup_account(platform)
          query: SELECT * FROM platform_accounts WHERE platform=? AND is_primary=false AND status='active'
        - step: 切换账号
          if: backup_account found
          then:
            - function: update_task_account(task_id, backup_account_id)
            - function: update_task_status(task_id, "failed_over")
            - function: retry_publish(task_id)
          else:
            - function: mark_task_failed(task_id, "no_backup_account")
            - function: send_alert("account_banned_no_backup", { platform, account_id })

    wait_random_interval:
      function: sleep(random.uniform(min_seconds, max_seconds))
      default_min: 180
      default_max: 600
```
