# M-23 跨平台热点追踪

---

## 1. 模块定位

| 属性 | 值 |
|------|-----|
| **模块编号** | M-23 |
| **模块名称** | 跨平台热点追踪 |
| **所属分类** | 数据分析 |
| **优先级** | P2（重要模块，为内容策划提供热点方向） |
| **触发方式** | 定时任务（每日 08:00 / 12:00 / 18:00 UTC+8，每日三次）；支持手动触发 |
| **在系统中的位置** | 从各平台热搜/热榜 API 采集热点数据，结合账号垂直领域标签进行相关性评分与趋势判定，输出可借势话题清单，供 M-24 下期计划草案生成使用 |

### 1.1 核心职责

1. **热点采集**：从抖音、小红书、B站、微博 4 个平台的热榜 API 采集热搜关键词和话题数据。
2. **领域过滤**：根据账号的垂直领域标签（如"科技"、"AI"、"效率工具"），筛选与账号定位相关的热点。
3. **趋势判定**：追踪热点的排名变化，识别上升趋势（可借势）和下降趋势（应规避）。
4. **相关性排序**：综合热度、趋势、领域相关性三个维度，对热点进行排序。
5. **可借势话题清单**：输出排序后的热点话题列表，包含建议的内容切入角度。

### 1.2 上下游关系

```
各平台热搜 API → M-23(本模块) → M-24(下期计划草案生成)
```

- **上游**：各平台的公开热榜/热搜 API（或爬虫数据源）
- **下游**：M-24 消费可借势话题清单，用于生成下期内容计划

## 4. 核心逻辑

### 4.1 主流程

```
开始
  │
  ├─ 1. 热点数据采集
  │     ├─ 并行调用各平台热搜 API
  │     ├─ 抖音：抖音热搜榜 API
  │     ├─ 小红书：小红书发现页热榜
  │     ├─ B站：B站热门话题/每周必看
  │     └─ 微博：微博热搜榜
  │     ├─ 各平台数据格式标准化
  │     └─ 去重合并（同一关键词出现在多平台）
  │
  ├─ 2. 热度归一化
  │     ├─ 各平台热度指数尺度不同
  │     ├─ 对每个平台的热度指数进行 Min-Max 归一化：
  │     │   normalized = (value - min) / (max - min)
  │     ├─ 排名转换为热度得分：
  │     │   rank_score = 1 / sqrt(rank)  # 排名越高得分越高
  │     └─ hot_score = 0.5 × normalized_hot_index + 0.5 × rank_score
  │
  ├─ 3. 领域相关性计算
  │     ├─ 加载账号 niche_tags
  │     ├─ 对每个热点关键词：
  │     │   ├─ 与每个标签进行匹配
  │     │   ├─ 匹配方式：
  │     │   │   ├─ exact：完全匹配（最高权重）
  │     │   │   ├─ keyword_overlap：关键词重叠
  │     │   │   └─ semantic：语义相似度（LLM 辅助）
  │     │   ├─ relevance_score = Σ(tag.weight × tag_category_weight × match_strength)
  │     │   ├─ IF relevance_score < min_relevance_score → 过滤
  │     │   └─ 记录 matched_tags
  │     └─ 负面关键词过滤：
  │         ├─ 如果热点匹配到 negative_keywords → 降低 relevance_score
  │         └─ 如果 negative_keyword 匹配强度 > 0.8 → 直接过滤
  │
  ├─ 4. 趋势判定
  │     ├─ 查询同一热点在上一快照中的数据
  │     ├─ 计算趋势指标：
  │     │   ├─ rank_change = previous_rank - current_rank
  │     │   │   （正=排名上升，注意排名越小越热）
  │     │   ├─ hot_index_change_pct = (current - previous) / previous × 100
  │     │   └─ momentum = normalize(rank_change) × 0.5 + normalize(hot_index_change_pct) × 0.5
  │     ├─ 趋势分类：
  │     │   ├─ IF momentum > 0.15 → "rising"（上升趋势，适合借势）
  │     │   ├─ IF momentum < -0.15 → "declining"（下降趋势，建议规避）
  │     │   ├─ IF 无历史数据 → "new_entry"（新上榜）
  │     │   ├─ IF 不在当前榜但历史上榜 → "exited"（已下榜）
  │     │   └─ ELSE → "stable"（稳定）
  │     └─ 过滤 declining 热点（根据配置）
  │
  ├─ 5. 跨平台检测
  │     ├─ 对相同或高度相似的关键词进行跨平台匹配
  │     ├─ 语义相似度 > 0.85 → 视为同一热点
  │     ├─ 记录 appearing_platforms 列表
  │     └─ cross_platform_score = len(appearing_platforms) / 4 × 0.2
  │         （出现在全部 4 个平台得最高分 0.2）
  │
  ├─ 6. 综合评分
  │     ├─ composite_score =
  │     │     hot_score × 0.3 +
  │     │     relevance_score × 0.4 +
  │     │     momentum_score × 0.3 +
  │     │     cross_platform_score（加成项，最多 +0.2）
  │     ├─ 按 composite_score 降序排序
  │     ├─ 取前 max_topics 条
  │     └─ 生成优先级：
  │         ├─ composite_score > 0.8 → P0_immediate
  │         ├─ composite_score > 0.6 → P1_high
  │         ├─ composite_score > 0.4 → P2_medium
  │         └─ composite_score ≤ 0.4 → P3_low
  │
  ├─ 7. 切入角度生成
  │     ├─ 对每个可借势话题：
  │     │   ├─ LLM 分析话题特征
  │     │   ├─ 结合账号领域标签
  │     │   ├─ 生成 1-3 个建议切入角度
  │     │   ├─ 建议内容形式
  │     │   └─ 评估时效性要求
  │     └─ 风险检测：
  │         ├─ 检查话题是否涉及敏感内容
  │         ├─ 检查是否有负面关联
  │         └─ 生成 risk_warnings
  │
  ├─ 8. 输出结果
  │     ├─ 写入 hot_topics 表
  │     ├─ 生成可借势话题清单
  │     ├─ 生成规避话题列表
  │     └─ 通知 M-24 热点数据已更新
  │
  └─ 9. 历史数据更新
        └─ 将本次快照存入历史热点表，供下次趋势判定
```

### 4.2 相关性计算算法

```
Function calculate_relevance(keyword, niche_tags):
    total_score = 0
    max_possible = 0
    matched_tags = []

    FOR each tag in niche_tags:
        category_weight = CATEGORY_WEIGHTS[tag.category]
        max_possible += tag.weight * category_weight

        # 精确匹配
        IF keyword == tag.tag:
            match_strength = 1.0
            match_method = "exact"
        # 关键词重叠
        ELIF has_keyword_overlap(keyword, tag.tag):
            match_strength = calculate_overlap_ratio(keyword, tag.tag)
            match_method = "keyword_overlap"
        # 语义相似度
        ELIF semantic_similarity(keyword, tag.tag) > 0.7:
            match_strength = semantic_similarity(keyword, tag.tag)
            match_method = "semantic"
        ELSE:
            CONTINUE

        # 检查负面关键词
        IF tag.negative_keywords:
            FOR neg_kw in tag.negative_keywords:
                IF neg_kw in keyword:
                    match_strength *= 0.3  # 降权
                    IF match_strength > 0.8:  # 强负面匹配
                        match_strength = 0  # 直接排除

        score = tag.weight * category_weight * match_strength
        total_score += score

        matched_tags.append({
            "tag": tag.tag,
            "category": tag.category,
            "match_strength": match_strength,
            "match_method": match_method
        })

    relevance_score = total_score / max_possible IF max_possible > 0 ELSE 0
    RETURN relevance_score, matched_tags
```

### 4.3 趋势动量计算

```
Function calculate_momentum(current_item, previous_snapshot):
    IF previous_snapshot is None:
        RETURN 0, "new_entry", 0.5

    previous = find_in_snapshot(previous_snapshot, current_item.item_id)

    IF previous is None:
        RETURN 0, "new_entry", 0.5

    # 排名变化（注意：排名越小越热，所以 previous - current > 0 表示上升）
    rank_change = previous.rank - current_item.current_rank
    rank_momentum = tanh(rank_change / 10)  # 归一化到 [-1, 1]

    # 热度指数变化
    IF previous.hot_index > 0:
        hot_index_change = (current_item.hot_index - previous.hot_index) / previous.hot_index
    ELSE:
        hot_index_change = 0
    hot_momentum = tanh(hot_index_change * 2)  # 归一化

    # 综合动量
    momentum = rank_momentum * 0.5 + hot_momentum * 0.5

    # 判定趋势方向
    IF momentum > 0.15:
        direction = "rising"
        confidence = min(0.9, 0.5 + abs(momentum))
    ELIF momentum < -0.15:
        direction = "declining"
        confidence = min(0.9, 0.5 + abs(momentum))
    ELSE:
        direction = "stable"
        confidence = 0.6

    RETURN momentum, direction, confidence
```

### 4.4 跨平台聚合

```
Function aggregate_cross_platform(all_hot_items):
    # 按关键词分组
    groups = {}

    FOR item in all_hot_items:
        matched = False
        FOR key in groups:
            IF semantic_similarity(item.keyword, key) > 0.85:
                groups[key].append(item)
                matched = True
                BREAK
        IF NOT matched:
            groups[item.keyword] = [item]

    # 合并跨平台数据
    merged = []
    FOR keyword, items in groups:
        platforms = [item.platform for item in items]
        avg_hot_score = mean([item.hot_score for item in items])
        avg_momentum = mean([item.momentum for item in items])

        merged.append({
            "keyword": keyword,
            "appearing_platforms": platforms,
            "is_cross_platform": len(platforms) > 1,
            "cross_platform_score": len(platforms) / 4 * 0.2,
            "avg_hot_score": avg_hot_score,
            "avg_momentum": avg_momentum
        })

    RETURN merged
```

## 7. 自学习机制

### 7.1 相关性阈值自适应

```
每月分析热点采纳率：
- 采纳率 = 被实际用于内容创作的热点 / 推荐的热点总数
- IF 采纳率 < 10%：
  → min_relevance_score 可能过高，过滤掉了有效热点
  → 自动降低 0.05（最低不低于 0.15）
- IF 采纳率 > 50%：
  → min_relevance_score 可能过低，推荐了不相关的热点
  → 自动提高 0.05（最高不超过 0.5）
```

### 7.2 趋势预测模型

```
记录每个热点的完整生命周期：
- 首次上榜时间
- 峰值排名和热度
- 下榜时间
- 总在榜时长

基于历史数据训练：
- 预测新上榜热点的可能峰值
- 预测热点在榜时长
- 提升对 rising 热点的早期识别能力
```

### 7.3 切入角度效果反馈

```
运营人员使用推荐切入角度后：
- 记录该角度的实际内容表现（从 M-18/M-20 获取）
- 如果内容表现好（engagement_rate > 基准）：
  → 该切入角度模板权重 +10%
- 如果内容表现差（engagement_rate < 基准）：
  → 该切入角度模板权重 -10%
- 定期淘汰表现持续差的切入角度模板
```

### 9.1 标准热点追踪指令

```
你作为 M-23 跨平台热点追踪模块的 Hermes Agent，请按照以下步骤执行热点追踪：

1. 热点数据采集：
   - 并行调用抖音、小红书、B站、微博的热搜/热榜 API
   - 采集每个平台 TOP 50 热搜条目
   - 标准化各平台数据格式
   - 记录采集时间戳和 fetch_id

2. 热度归一化：
   - 对每个平台的热度指数单独进行 Min-Max 归一化
   - 排名转换为得分：rank_score = 1 / sqrt(rank)
   - 综合热度：hot_score = 0.5 × normalized_hot_index + 0.5 × rank_score
   - 计算每个热点在同平台的百分位排名

3. 领域相关性计算：
   - 加载目标账号的 niche_tags 配置
   - 对每个热点关键词与所有标签进行匹配：
     a. 精确匹配（最高权重）
     b. 关键词重叠（中等权重）
     c. 语义相似度（LLM 辅助，低权重）
   - 考虑标签类别权重：core × 1.0, extended × 0.6, tangential × 0.3
   - 检查负面关键词，匹配到则降权或排除
   - 过滤 relevance_score < 0.3 的热点

4. 趋势判定：
   - 从 hot_snapshots 表查询上一次快照（24h 内）
   - 对每个热点计算：
     a. rank_change = previous_rank - current_rank（正值=排名上升）
     b. hot_index_change_pct
     c. momentum = normalized_rank_change × 0.5 + normalized_hot_index_change × 0.5
   - 判定趋势方向：
     - momentum > 0.15 → rising
     - momentum < -0.15 → declining
     - 无历史数据 → new_entry
     - 否则 → stable
   - 根据配置决定是否过滤 declining 热点

5. 跨平台检测：
   - 对语义相似度 > 0.85 的关键词进行跨平台合并
   - 计算 cross_platform_score
   - 标记 is_cross_platform

6. 综合评分与排序：
   - composite_score = hot_score × 0.3 + relevance_score × 0.4 + momentum_score × 0.3 + cross_platform_score
   - 按 composite_score 降序排序
   - 取前 20 条作为可借势话题

7. 切入角度生成：
   - 对每个可借势话题调用 LLM 生成 1-3 个建议切入角度
   - 每个角度包含：内容形式建议、时效性要求
   - 检测风险：话题敏感度、负面关联
   - 生成 risk_warnings

8. 输出结果：
   - 写入 hot_topics 表
   - 生成可借势话题清单 → actionable_topic_lists 表
   - 生成规避话题列表（declining + 高风险话题）
   - 更新 hot_snapshots 表（保存本次快照）

9. 通知 M-24 热点数据已更新

注意事项：
- 各平台 API 调用频率需控制在限额内
- 跨平台语义匹配阈值需定期校准
- 切入角度需结合账号定位，避免泛泛而谈
- 风险检测需严格，宁可多报不可漏报
```
