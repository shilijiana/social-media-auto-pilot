# Hermes 系统架构概览

> **项目名称**: Social Media Auto-Pilot（社交媒体自动化运营系统）
> **代号**: Hermes
> **目标**: 构建人机协同、具备自学习能力的社交媒体自动化运营闭环
> **覆盖平台**: 抖音、小红书、B站、X（Twitter）
> **文档受众**: 开发人员、架构师、Hermes Agent

---

## 1. 项目概述

Hermes 是一个一站式的多平台社交媒体自动化运营引擎，覆盖从素材采集、内容生产、审阅管理、多平台分发、数据分析到自学习的全链路。系统采用模块化架构，共 30 个独立模块，划分为 6 大服务域。

### 1.1 核心能力

- **4 平台覆盖**: 抖音、小红书、B站、X（Twitter）
- **6 大服务域**: 素材采集、内容生产、审阅管理、分发采集、数据分析、自学习
- **30 个模块**: 每个模块独立开发、独立测试、独立部署
- **3 阶段演进**: P0 核心链路 → P1 质量增强 → P2/P3 智能进化

### 1.2 核心闭环

```
│       │             数据分析 ◄────────────────────────┘          │

│  5. 数据分析 → 自学习: 分析结果驱动偏好学习和规则优化            │
│  6. 自学习 → 素材采集: 学习成果反馈到关键词扩展和选题方向        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **Web 框架** | FastAPI | 高性能异步 Web 框架，提供 RESTful API |
| **ORM** | SQLAlchemy 2.0 | 数据库 ORM，支持异步操作 |
| **数据库** | SQLite (开发) / PostgreSQL (生产) | 轻量级开发，生产环境切换 PostgreSQL |
| **数据校验** | Pydantic v2 | 请求/响应数据校验和序列化 |
| **浏览器自动化** | Playwright | 跨平台浏览器自动化，用于数据采集和内容发布 |
| **AI/LLM** | OpenAI API 兼容接口 | 文案生成、评论分析、热点识别 |
| **媒体处理** | FFmpeg | 视频转码、剪辑、封面提取 |
| **任务调度** | Celery + Redis | 异步任务队列和定时任务调度 |
| **配置管理** | YAML + 环境变量 | 配置文件和环境变量双通道 |
| **日志** | Loguru | 结构化日志输出 |
| **容器化** | Docker + Docker Compose | 统一运行环境 |

## 4. 六大服务域

### 4.1 素材采集域 (Material Ingestion Domain)

**职责**: 从外部平台和渠道获取、清洗、去重和入库各类素材资源。

| 模块 | 名称 | 优先级 | 说明 |
|------|------|--------|------|
| M-01 | 关键词智能扩展 | P2 | 基于 LLM 扩展搜索关键词 |
| M-02 | 图文素材采集 | P0 | 从各平台采集图片和文字素材 |
| M-03 | 视频素材采集 | P0 | 从各平台采集视频素材 |
| M-04 | 参考文案与竞品采集 | P2 | 采集竞品内容和参考文案 |
| M-05 | 素材去重与质量控制 | P1 | 去重、清晰度检测、质量评分 |
| M-06 | 素材入库与版权追踪 | P1 | 素材入库、标签管理、版权状态跟踪 |

### 4.2 内容生产域 (Content Production Domain)

**职责**: 基于已入库素材进行内容创作，包括文案、封面、视频的全链路生产。

| 模块 | 名称 | 优先级 | 说明 |
|------|------|--------|------|
| M-07 | 素材匹配与调用 | P0 | 根据选题匹配合适的素材 |
| M-08 | 文案差异化生成 | P0 | LLM 驱动多平台文案生成 |
| M-09 | 封面图生成 | P0 | AI 封面图生成与模板合成 |
| M-10 | 视频初剪 | P0 | 视频自动剪辑、转场、字幕 |
| M-11 | 内容精加工与定稿 | P0 | 内容最终润色和定稿输出 |
| M-12 | 合规性自检 | P1 | 敏感词、广告法、版权检查 |

### 4.3 审阅管理域 (Review Management Domain)

**职责**: 对生产出的内容进行分级审阅、版本变更追踪和通用素材池管理。

| 模块 | 名称 | 优先级 | 说明 |
|------|------|--------|------|
| M-13 | 分级审阅引擎 | P1 | A/B/C 三级审阅模式路由 |
| M-14 | 版本变更管理 | P1 | 内容版本追踪和变更历史 |
| M-15 | 通用素材池管理 | P2 | 跨内容复用的通用素材管理 |

### 4.4 分发采集域 (Distribution & Collection Domain)

**职责**: 将定稿内容分发到各目标平台，监控发布状态，采集效果数据和账号健康度。

| 模块 | 名称 | 优先级 | 说明 |
|------|------|--------|------|
| M-16 | 多平台分发 | P0 | 一键分发到抖音/小红书/B站/X |
| M-17 | 发布状态监控与应急响应 | P0 | 发布状态追踪、失败重试、应急处理 |
| M-18 | 效果数据采集 | P0 | 采集播放量、互动数据等效果指标 |
| M-19 | 账号健康度监控 | P1 | 粉丝趋势、违规记录、限流检测 |

### 4.5 数据分析域 (Data Analytics Domain)

**职责**: 对采集到的效果数据进行清洗、聚合和多维度分析，挖掘评论语义，追踪热点。

| 模块 | 名称 | 优先级 | 说明 |
|------|------|--------|------|
| M-20 | 数据清洗与聚合 | P2 | 缺失值处理、异常检测、维度聚合 |
| M-21 | 多维度效果分析 | P2 | 六大维度综合分析 |
| M-22 | 评论区语义挖掘与隐私脱敏 | P1 | NLP 情感分析、主题聚类、PII 脱敏 |
| M-23 | 跨平台热点追踪 | P2 | 多平台热搜监控和趋势预测 |
| M-24 | 下期计划草案生成 | P2 | 综合分析结果生成选题和策略建议 |

### 4.6 自学习域 (Self-Learning Domain)

**职责**: 从运营数据中持续学习，提炼用户偏好和运营规则，反馈到内容生产和分发环节。

| 模块 | 名称 | 优先级 | 说明 |
|------|------|--------|------|
| M-25 | 学习数据收集 | P1 | 收集审阅反馈和效果数据 |
| M-26 | 偏好提炼与规则生成 | P1 | 从数据中提炼偏好规则 |
| M-27 | 规则时效性管理 | P3 | 规则生命周期管理 |
| M-28 | 初始偏好模板导入 | P3 | 导入预设偏好模板 |
| M-29 | 学习成果应用 | P2 | 规则分发到各业务模块 |
| M-30 | 学习成效量化与报告 | P3 | 评估学习效果并生成报告 |

## 6. 三阶段演进

### 6.1 P0 — 核心链路（人工主导期，第 1-4 周）

**目标**: 打通"采集 → 生产 → 分发 → 效果回收"的最短闭环。

**模块范围**: M-02, M-03, M-07, M-08, M-09, M-10, M-11, M-16, M-17, M-18（共 10 个模块）

```
P0 链路:
M-02/M-03(素材采集) → M-07(素材匹配) → M-08(文案生成)
    → M-09(封面生成) → M-10(视频初剪) → M-11(精加工定稿)
    → M-16(多平台分发) → M-17(状态监控) → M-18(效果采集)
```

**验收标准**:
- 单平台（抖音）完整链路可运行
- 内容可自动生产并发布
- 发布后可采集基础效果数据

### 6.2 P1 — 质量增强（人机协同期，第 5-12 周）

**目标**: 增强内容质量管控、审阅流程和账号安全，引入初步自学习能力。

**模块范围**: M-05, M-06, M-12, M-13, M-14, M-19, M-22, M-25, M-26（共 9 个模块）

```
P1 新增:
M-05(去重质控) + M-06(版权追踪) → 素材质量保障
M-12(合规自检) + M-13(分级审阅) + M-14(版本管理) → 内容质量管控
M-19(账号健康) + M-22(评论挖掘) → 数据深度分析
M-25(学习收集) + M-26(规则生成) → 初步自学习
```

**验收标准**:
- 四平台全链路可运行
- 审阅流程完整可用
- 基础偏好规则可生成

### 6.3 P2/P3 — 智能进化（系统主导期，第 13 周起）

**目标**: 实现数据分析驱动的策略优化和完整的自学习闭环。

**模块范围**: M-01, M-04, M-15, M-20, M-21, M-23, M-24, M-27, M-28, M-29, M-30（共 11 个模块）

```
P2/P3 新增:
M-01(关键词扩展) + M-04(竞品采集) → 智能素材采集
M-20(数据清洗) + M-21(多维分析) + M-23(热点追踪) + M-24(计划生成) → 智能分析
M-15(通用素材池) → 资源复用
M-27(时效管理) + M-28(模板导入) + M-29(成果应用) + M-30(成效报告) → 完整自学习
```

**验收标准**:
- 全 30 模块完整运行
- 自学习闭环生效（人工修改率持续下降）
- 系统具备自我优化能力

## 9. 关键设计原则

### 9.1 模块独立性

- 每个模块独立开发、独立测试、独立部署
- 模块间通过明确的 JSON Schema 接口通信
- 模块内部实现细节不暴露给外部

### 9.2 数据驱动

- 所有配置通过 YAML 文件和数据库管理
- 规则和偏好从数据中自动提炼，减少硬编码
- 系统行为由数据反馈驱动，而非预设规则

### 9.3 渐进式演进

- P0 先跑通最小闭环，验证核心链路可行性
- P1 增加质量管控和审阅流程
- P2/P3 实现智能分析和自学习

### 9.4 人机协同

- 关键决策保留人工审核环节
- 系统提供建议和辅助，最终决策权归人
- 从人工决策中学习，逐步提升自动化程度
